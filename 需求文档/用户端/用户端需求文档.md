# 用户端需求文档（细化版）

## 1. 文档目标
用于指导用户端功能开发、联调与测试验收，覆盖页面功能、交互逻辑、字段展示规则与权限分支。

**版本**：v2.0（开发同步版）
**更新日期**：2026-06-05

## 1.1 当前开发同步说明

截至 2026-06-05，用户端已开发以下页面：

| 模块 | 路由 / 页面 | 已实现能力 |
|---|---|---|
| 商品浏览 | 商品列表、商品详情 | 仅展示已上架商品；搜索、热搜、历史、角色/IP筛选；详情不可见时展示失效态 |
| 我的 | 我的首页、修改密码 | 用户资料入口、寄售入口、退出登录、修改密码 |
| 我的寄售 | 说明页、寄售管理首页、寄售商品、上传、已出售、已结算、结算详情 | 按寄售权限分流；寄售用户可查看自己的商品、售出和结算记录 |

## 2. 页面与功能清单
- 商品列表页
- 商品详情页
- 搜索功能（列表页内）
- 我的页面
- 修改密码页
- 我的寄售（权限分支）
- 寄售管理页
- 寄售商品页
- 上传商品页
- 已出售页
- 已结算页

## 3. 商品列表页

### 3.1 页面目标
向普通用户展示可购买商品，并提供筛选和搜索能力。

### 3.2 数据范围与默认规则
- 仅拉取并展示“上架”状态商品。
- 页面初次进入时自动加载第一页数据。
- 若无数据，显示空状态文案：暂无上架商品。

### 3.3 字段展示规则
- 不显示字段：
  - 商品状态
  - 寄售用户昵称
- 其他商品字段全部展示（如商品图、名称、价格、角色、系列、IP、库存、发布时间等，具体以后端返回为准）。

### 3.4 筛选功能
- 当前已实现筛选项：角色、IP。
- 系列筛选暂未开发，如后续需要可作为二期补充。
- 支持单项筛选与多项组合筛选。
- 用户点击“重置”后恢复默认状态（即仅“上架商品”条件生效）。
- 每次筛选条件变化后，重置分页并重新请求数据。

### 3.5 列表交互
- 点击商品卡片跳转商品详情页。
- 支持下拉刷新（重置到第一页）。
- 支持上拉加载更多（若无更多则提示“没有更多了”）。
- 接口异常时显示加载失败提示，并支持重试。

## 4. 商品详情页

### 4.1 页面目标
展示用户查看商品并决定购买所需的信息。

### 4.2 字段展示规则
- 不显示字段：
  - 寄售用户
  - 抽成比例
  - 已结算
- 商品状态需要显示。
- 其他字段均显示（如商品图、名称、价格、描述、IP、角色、系列、库存、发货信息等，具体以接口返回为准）。

### 4.3 交互功能
- 支持商品主图预览（点击放大查看）。
- 商品不存在、已下架或不可见时，显示失效状态并提供返回列表入口。

## 5. 搜索功能

### 5.1 搜索入口
- 搜索框位于商品列表页顶部。

### 5.2 搜索范围
- 关键词匹配范围：IP、角色、系列。

### 5.3 搜索规则
- 输入关键词后点击搜索触发查询。
- 清空关键词后恢复默认列表（仅上架商品）。
- 搜索条件可与筛选条件叠加。
- 无搜索结果时显示空状态文案：未找到相关商品。

### 5.4 搜索历史与热搜
- 当前已实现搜索历史，最多保存最近 6 条关键词。
- 支持一键清空历史。
- 当前已实现热搜词：咒术回战、BLUE LOCK、间谍过家家、原神、崩坏：星穹铁道、排球少年。
- 点击历史词或热搜词后填入搜索框并触发搜索。

## 6. 我的页面

### 6.1 页面目标
提供用户基础信息入口、账户设置入口、寄售业务入口。

### 6.2 功能模块
- 个人信息展示：头像、昵称。
- 修改密码入口。
- 我的寄售入口。

### 6.3 交互逻辑
- 点击“修改密码”进入修改密码页（见第7节）。
- 点击“我的寄售”根据寄售权限进入不同页面（见第8节）。

## 7. 修改密码页

### 7.1 页面目标
用户可在登录状态下修改当前账号密码，提升账号安全性。

### 7.2 页面样式
- 页面标题：修改密码。
- 页面整体采用表单布局，内容纵向排列。
- 表单项包含左侧字段名称和右侧输入框，输入框使用密码输入类型。
- 输入框需支持明文/密文切换图标，默认密文显示。
- 页面底部固定或随内容展示“确认修改”按钮。
- “确认修改”按钮在必填项未完整填写或校验不通过时置灰不可点击。
- 提交中按钮显示加载状态，避免重复提交。

### 7.3 表单字段
- 原密码：必填，用于校验当前账号密码。
- 新密码：必填，用于设置新密码。
- 确认新密码：必填，需要与新密码保持一致。

### 7.4 校验规则
- 原密码不能为空。
- 新密码不能为空。
- 确认新密码不能为空。
- 新密码长度建议为6-20位。
- 新密码不能与原密码一致。
- 确认新密码必须与新密码一致。
- 输入框失焦时可进行单项校验，点击“确认修改”时需进行全量校验。

### 7.5 提示文案
- 原密码为空：请输入原密码。
- 新密码为空：请输入新密码。
- 新密码长度不符合要求：新密码长度需为6-20位。
- 新密码与原密码一致：新密码不能与原密码相同。
- 确认新密码为空：请再次输入新密码。
- 两次新密码不一致：两次输入的新密码不一致。
- 修改成功：密码修改成功，请重新登录。
- 原密码错误：原密码错误，请重新输入。
- 修改失败：修改失败，请稍后重试。

### 7.6 提交流程
- 用户填写原密码、新密码、确认新密码。
- 前端完成必填、长度、一致性校验。
- 校验通过后调用修改密码接口。
- 修改成功后清除登录状态，并跳转登录页。
- 修改失败时停留当前页面并展示错误提示。

### 7.7 安全规则
- 密码输入框默认不展示明文。
- 密码内容不在页面、日志、提示文案中明文展示。
- 修改成功后需要重新登录。
- 接口提交时仅提交必要字段：原密码、新密码。

## 8. 我的寄售（权限分支）

### 8.1 未开启寄售权限用户
点击“我的寄售”后进入“寄售说明页”，展示：
- 寄售介绍说明（寄售流程、结算说明、注意事项）。
- 客服联系方式（微信号/手机号/二维码，按运营配置展示）。

### 8.2 已开启寄售权限用户
点击“我的寄售”后进入“寄售管理页”。

## 9. 寄售管理页

### 9.1 页面内容
- 抽成比例（当前用户寄售抽成比例）。
- 寄售商品入口。
- 已出售入口。
- 已结算入口。

### 9.2 页面交互
- 点击“寄售商品入口”跳转寄售商品页。
- 点击“已出售入口”跳转已出售页。
- 点击“已结算入口”跳转已结算页。

## 10. 寄售商品页

### 10.1 数据范围
- 显示当前登录用户的全部寄售商品。

### 10.2 展示与交互
- 页面右上角或列表顶部展示“添加”按钮。
- 点击“添加”按钮后打开上传商品页。
- 按时间倒序展示（最新寄售在前）。
- 支持查看商品状态标签（如上架、已出售、下架等）。
- 点击商品信息区域跳转对应的商品详情页。
- 商品状态为“已下架”时，用户可以编辑和删除该商品。
- 商品状态为“上架”“已出售”“已结算”时，不提供编辑和删除操作。
- 点击“编辑”进入商品编辑页，编辑页样式和字段与上传商品页保持一致，并回填当前商品信息。
- 点击“删除”需二次确认，确认后删除商品，删除成功后刷新寄售商品列表。

### 10.3 空状态
- 当前用户暂无寄售商品时显示空状态。
- 空状态文案：暂无寄售商品。
- 空状态页面保留“添加商品”入口，点击后进入上传商品页。

## 11. 上传商品页

### 11.1 页面目标
已开启寄售权限的用户可自行上传寄售商品，上传后的商品自动归属当前登录用户。

### 11.2 页面样式
- 页面样式与管理端“上传商品页”保持一致。
- 页面结构、表单布局、图片上传样式、字段顺序、必填标识、提交按钮样式与管理端一致。
- 用户端不展示“选择寄售用户”字段。

### 11.3 字段规则
- 商品照片，默认只显示添加按钮，最多9张，九宫格形式。
- IP，必填，独占一行。
- 角色，必填，独占一行。
- 系列，必填，独占一行。
- 价格，必填，独占一行。
- 类型，必填，下拉框。
- 自定义类型输入框，仅类型选择“自定义”时显示。
- 数量，必填。
- 质量，必填，分段选择：无暇 / 有瑕。
- 备注。
- 提交上传按钮，直接跟随表单内容显示，不使用底部遮罩层。
- 当前用户端上传不展示购买记录字段，系统默认保存为"无"。

### 11.4 业务规则
- 上传商品时系统自动绑定当前登录用户为寄售用户。
- 用户端上传商品无需选择寄售用户，也不可代其他用户上传。
- 用户端上传商品无需填写商品状态。
- 用户提交商品后，商品状态统一默认为“已下架”。
- 用户端上传商品不填写购物平台链接；购物平台链接由管理员在管理端补充后上架。
- 必填项字段名称后面显示红色“*”。
- 点击提交前需完成必填校验。
- 上传成功后返回寄售商品页并刷新列表。

## 12. 已出售页

### 12.1 数据范围
- 显示当前用户已出售商品。
- 数据仅包含当前登录用户名下的寄售商品。
- 商品状态标签为“已出售”。

### 12.2 页面样式
- 页面样式与管理端“已出售商品”页面保持一致。
- 列表结构、商品信息展示、状态标签、金额信息、时间信息与管理端已出售商品页面对齐。

### 12.3 展示与交互
- 默认按出售时间倒序展示。
- 可展示出售价格、出售时间、订单编号（以接口字段为准）。

### 12.4 空状态
- 当前用户暂无已出售商品时显示空状态。
- 空状态文案：暂无已出售商品。
- 空状态不展示商品列表，不展示分页加载。

## 13. 已结算页

### 13.1 数据范围
- 显示当前用户已结算的商品。
- 数据仅包含当前登录用户名下的寄售商品。
- 数据口径与管理端“已结算商品”页面一致。

### 13.2 页面样式
- 页面样式与管理端“已结算商品”页面保持一致。
- 列表结构、商品信息展示、结算信息、金额信息、时间信息与管理端已结算商品页面对齐。

### 13.3 展示与交互
- 显示结算金额、结算时间、结算单号/批次号（以接口字段为准）。
- 支持按时间倒序展示。

### 13.4 空状态
- 当前用户暂无已结算商品时显示空状态。
- 空状态文案：暂无已结算商品。
- 空状态不展示商品列表，不展示分页加载。

## 14. 路由与跳转关系
- 我的页面 -> 修改密码 -> 修改密码页
- 我的页面 -> 我的寄售 ->
  - 未开通权限：寄售说明页（介绍说明 + 客服联系方式）
  - 已开通权限：寄售管理页
- 寄售管理页 ->
  - 寄售商品入口 -> 寄售商品页
  - 已出售入口 -> 已出售页
  - 已结算入口 -> 已结算页
- 寄售商品页 -> 添加 -> 上传商品页
- 寄售商品页 -> 点击商品信息 -> 商品详情页
- 寄售商品页 -> 已下架商品 -> 编辑 -> 商品编辑页
- 商品列表页 -> 点击商品卡片 -> 商品详情页

## 15. 状态与异常处理
- 列表类页面统一包含：加载中、空状态、加载失败三种状态。
- 网络异常时统一提示：网络异常，请稍后重试。
- 接口返回无权限时，寄售相关页面提示无权限并提供联系客服入口。

- 商品列表仅出现上架商品。
- 商品列表不显示“商品状态、寄售用户昵称”。
- 商品详情不显示“寄售用户、抽成比例、已结算”。
- 商品详情需要显示商品状态。
- 搜索按 IP / 角色 / 系列生效，且可与角色 / IP 筛选叠加。
- 修改密码页包含原密码、新密码、确认新密码三个必填项。
- 修改密码需校验必填、长度、新旧密码不同、两次新密码一致。
- 修改密码成功后清除登录状态并跳转登录页。
- 我的寄售按权限正确分流。
- 已开启权限用户可进入寄售管理，并可跳转三个子页面。
- 寄售商品页范围正确（当前用户全部寄售）。
- 寄售商品页无数据时显示“暂无寄售商品”，并保留添加商品入口。
- 寄售商品列表点击商品信息可跳转对应商品详情页。
- 寄售商品页点击“添加”可进入上传商品页。
- 用户端上传商品页与管理端上传商品页样式一致，且不显示“选择寄售用户”字段。
- 用户端上传商品页不显示商品状态字段。
- 用户端上传商品页不显示购物平台链接字段。
- 用户端上传商品页不显示购买记录字段，系统默认保存为"无"。
- 用户提交商品后默认状态为“已下架”。
- 用户端上传商品自动绑定当前登录用户为寄售用户。
- 仅“已下架”状态商品可编辑和删除，其他状态不展示编辑和删除操作。
- 已出售页范围正确（当前用户已出售商品），样式与管理端已出售商品页面一致。
- 已出售页无数据时显示“暂无已出售商品”。
- 已结算页范围正确（当前用户已结算商品），样式与管理端已结算商品页面一致。
- 已结算页无数据时显示“暂无已结算商品”。
