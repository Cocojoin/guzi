# 用户端需求文档（细化版）

## 1. 文档目标
用于指导用户端功能开发、联调与测试验收，覆盖页面功能、交互逻辑、字段展示规则与权限分支。

**版本**：v2.2（功能优化补充版）  
**更新日期**：2026-06-16

## 1.1 当前需求同步说明

本次需求补充后，用户端“商品浏览”模块调整为 **IP 分类优先** 的结构：

| 模块 | 路由 / 页面 | 目标能力 |
|---|---|---|
| 商品浏览 | IP分类页、搜索页、IP商品列表页、商品详情页 | 登录后默认进入 IP分类页；先选 IP，再浏览对应商品；支持 IP 维度 Banner、独立搜索页和搜索结果列表 |
| 我的 | 我的首页、修改密码 | 用户资料入口、寄售入口、退出登录、修改密码 |
| 我的寄售 | 说明页、寄售管理首页、寄售商品、上传、已出售、已结算、结算详情 | 按寄售权限分流；寄售用户可查看自己的商品、售出和结算记录 |

## 3. IP分类页

### 3.1 页面目标
- 作为普通用户登录后的默认首页。
- 展示当前可浏览的 IP 分类，并提供全站上架商品搜索入口。

### 3.2 数据范围与默认规则
- 普通用户登录成功后，默认进入 IP分类页。
- 页面只统计和展示“已上架”状态商品对应的 IP。
- 同一 IP 下只要存在 1 个上架商品，即显示该 IP。
- 无上架商品的 IP 不在用户端显示。
- 若暂无上架商品，显示空状态文案：暂无可浏览 IP。

### 3.3 页面结构
- 顶部：页面标题“谷子集市”。
- 顶部搜索框：搜索全站上架商品。
- 中部 Banner：展示“最新上新”的 IP。
- 下部 IP 列表：展示全部可浏览 IP。

### 3.4 Banner 规则
- Banner 展示最近有“上架商品更新”的 IP。
- “最新上新”建议按商品 `updatedAt / createdAt` 倒序去重后取前 N 个 IP。
- Banner 至少展示：
  - IP 名称
  - 当前在售商品数
  - 当前上架角色数
  - 最新上新标签（如“今日上新 / 最新上新”）
- 点击 Banner，跳转对应 IP 的商品列表页。
- 若当前无可展示内容，则隐藏 Banner 区块。

### 3.5 IP 列表展示规则
- 每个 IP 卡片至少展示：
  - IP 名称
  - 当前在售商品数
  - 当前上架角色数
  - 右侧进入箭头
- “当前在售商品数”需作为每个 IP 的明确展示字段，不可隐藏。
- 可补充展示 IP 首字母/简称封面。
- 默认排序建议：
  1. 最近上新时间倒序
  2. 当前在售商品数倒序
  3. IP 名称正序

### 3.6 搜索入口规则
- 搜索框位于 IP分类页顶部。
- 点击搜索框进入独立搜索页 `user/pages/goods/search/search`。
- IP分类页本身不直接展示搜索结果。

### 3.7 页面交互
- 点击 Banner 进入对应 IP 商品列表页。
- 点击 IP 卡片进入对应 IP 商品列表页。
- 支持下拉刷新，刷新 Banner 和 IP 列表。
- 接口异常时显示加载失败提示，并支持重试。

## 4. IP商品列表页

### 4.1 页面目标
- 展示某一个 IP 下的全部上架商品。
- 或在“全站搜索结果模式”下，展示搜索命中的上架商品。

### 4.2 页面模式

#### 4.2.1 IP模式
- 页面必须带 IP 参数进入。
- 仅展示“当前选中 IP + 已上架状态”的商品。
- 页面标题显示当前 IP 名称。
- 无商品时显示空状态文案：该 IP 暂无上架商品。

#### 4.2.2 全站搜索结果模式
- 从搜索页执行搜索后进入。
- 展示全部上架商品中命中的结果。
- 页面不再作为默认首页，只作为搜索结果承载页。
- 页面标题显示“搜索结果”。

### 4.3 字段展示规则
- 不显示字段：
  - 寄售用户昵称
- 商品状态不作为主信息展示；用户进入该页默认看到的应为可浏览商品。
- 其他字段按现有商品卡规则展示，如商品图、角色、系列、价格、IP 等。

### 4.4 筛选规则
- 筛选项改为：角色、系列。
- 不再提供 IP 筛选项。
- 支持角色、系列组合筛选。
- 用户点击“重置”后恢复默认状态。
- 每次筛选条件变化后，立即刷新当前结果。

### 4.5 搜索规则

#### 4.5.1 IP模式
- 搜索范围仅限当前选中 IP 内的上架商品。
- 搜索匹配字段：角色、系列。
- 不跨 IP 搜索。

#### 4.5.2 全站搜索结果模式
- 搜索范围为全部上架商品。
- 搜索匹配字段：IP、角色、系列。

### 4.6 列表交互
- 点击商品卡片跳转商品详情页。
- 支持返回 IP分类页。
- 支持下拉刷新。
- 如启用分页，支持上拉加载更多；无更多时提示“没有更多了”。
- 接口异常时显示加载失败提示，并支持重试。

## 5. 商品详情页

### 5.1 页面目标
展示用户查看商品并决定购买所需的信息。

### 5.2 字段展示规则
- 不显示字段：
  - 寄售用户
  - 抽成比例
  - 已结算
- 商品状态需要显示。
- 其他字段均显示（如商品图、名称、价格、描述、IP、角色、系列、库存、发货信息等，具体以接口返回为准）。
- 商品详情页需展示“在哪里买”模块，用于告知用户可购买平台和店铺搜索方式，但**不展示商品外链 URL**。
- 商品详情页需展示“联系客服”模块，用于咨询商品详情、申请对光验货、售后帮助，以及向客服索取对应平台购买链接。

### 5.2.1 购买途径模块

模块标题为“在哪里买”，位于商品描述下方、联系客服模块上方。

| 元素 | 展示规则 |
|---|---|
| 模块标题 | “在哪里买” |
| 安全提示 | 右侧展示“认准官方店铺 · 谨防仿货” |
| 平台行 | 仅展示该商品已配置的购买平台；平台包括淘宝 / 天猫、拼多多、小红书 |
| 平台标识 | 使用平台短标识：淘、拼、书；颜色需与平台区分但保持页面整体风格 |
| 平台名称 | 展示“淘宝/天猫”“拼多多”“小红书” |
| 店铺名称 | 展示管理端“店铺信息设置”统一配置的店铺名，如“谷圈星社官方店”“星社の谷子铺” |
| 复制按钮 | 每个平台行右侧展示“复制”，点击复制店铺名或搜索关键词 |
| 底部说明 | 展示“复制店铺名后，到对应平台搜索店铺即可进店购买” |

🟩 展示口径：
- 用户端只展示购买平台、店铺名称、搜索关键词，不展示平台商品链接。
- 若同一商品配置多个平台，按淘宝/天猫、拼多多、小红书顺序展示。
- 若店铺名称与搜索关键词不同，页面主文案展示店铺名称，复制按钮优先复制搜索关键词。
- 若商品未配置任何购买途径，不展示“在哪里买”模块。
- 若商品已下架、售罄或不可见，不展示购买途径模块。

🟨 交互规则：
- 点击“复制”后复制对应平台的店铺名或搜索关键词，Toast “店铺名已复制”。
- 点击平台行本身不跳转外部平台。
- 不提供打开淘宝、拼多多、小红书的外链按钮。
- 复制失败时 Toast “复制失败，请手动搜索店铺名”。

### 5.2.2 联系客服模块

模块标题为“联系客服”，位于“在哪里买”模块下方。

| 元素 | 展示规则 |
|---|---|
| 图标 | 展示客服 / 耳机图标 |
| 标题 | “联系客服” |
| 说明 | “咨询商品详情、申请对光验货、售后都找客服” |
| 按钮 | “联系” |

🟨 交互规则：
- 点击“联系”唤起微信客服能力。
- 联系客服时需尽量携带当前商品上下文，便于客服发送购买链接或处理咨询。
- 建议传递内容：商品编号、商品名称、IP、角色、系列、用户当前选择的平台（如从某个平台行进入，则携带平台）。
- 客服可向用户发送对应平台商品链接；用户端商品详情页本身不展示链接。

### 5.3 交互功能
- 支持商品主图预览（点击放大查看）。
- 商品不存在、已下架或不可见时，显示失效状态并提供返回列表入口。
- 支持复制购买途径中的店铺名 / 搜索关键词。
- 支持点击“联系客服”进入客服会话。

## 6. 搜索功能（汇总）

### 6.1 搜索入口
- IP分类页顶部搜索框：进入独立搜索页。
- 搜索页：展示输入框、搜索历史、热门 IP、搜索结果。
- IP商品列表页顶部搜索框：搜索当前 IP 内上架商品或承载全站搜索结果二次搜索。

### 6.2 搜索范围
- 搜索页搜索：IP、角色、系列、类型。
- IP商品列表页搜索：
  - IP模式：角色、系列
  - 全站搜索结果模式：IP、角色、系列

### 6.3 搜索规则
- 输入关键词后点击搜索触发查询；搜索页先在当前页展示结果。
- 搜索页结果项点击后进入商品详情。
- 搜索结果列表页可通过 `mode=search&keyword=xxx` 承载全站搜索结果，支持角色 / 系列二次筛选。
- 清空关键词后恢复默认列表。
- 搜索条件可与筛选条件叠加。
- 无搜索结果时显示空状态文案：未找到相关商品。

### 6.4 搜索历史与热搜
- 搜索页保存最近 8 条搜索历史。
- 支持一键清空历史。
- 热门词基于上架商品 IP 聚合生成，最多展示 8 个。
- 热门排序优先级：用户本地搜索次数倒序 > 最近上新时间倒序 > IP 下商品数量倒序。
- 点击历史词或热搜词后填入搜索框并触发搜索。

### 6.5 IP分类页聚合规则
- IP分类页只读取已上架商品。
- IP 卡片的“在售商品数”按该 IP 下上架商品的剩余可售数量合计。
- IP 卡片的“角色数”按该 IP 下上架商品的角色去重数量计算。
- Banner 取排序后的前 3 个 IP。
- Banner 标签：若最近更新时间为当天，显示“今日上新”；否则显示“最新上新”。

## 7. 我的页面

### 7.1 页面目标
提供用户基础信息入口、账户设置入口、寄售业务入口。

### 7.2 功能模块
- 个人信息展示：头像、昵称。
- 修改密码入口。
- 我的寄售入口。

### 7.3 交互逻辑
- 点击“修改密码”进入修改密码页（见第8节）。

## 8. 修改密码页

### 8.1 页面目标
用户可在登录状态下修改当前账号密码，提升账号安全性。

### 8.2 页面样式
- 页面标题：修改密码。
- 页面整体采用表单布局，内容纵向排列。
- 表单项包含左侧字段名称和右侧输入框，输入框使用密码输入类型。
- 输入框需支持明文/密文切换图标，默认密文显示。
- 页面底部固定或随内容展示“确认修改”按钮。
- “确认修改”按钮在必填项未完整填写或校验不通过时置灰不可点击。
- 提交中按钮显示加载状态，避免重复提交。

### 8.3 表单字段
- 原密码：必填，用于校验当前账号密码。
- 新密码：必填，用于设置新密码。
- 确认新密码：必填，需要与新密码保持一致。

### 8.4 校验规则
- 原密码不能为空。
- 新密码不能为空。
- 确认新密码不能为空。
- 新密码长度建议为6-20位。
- 新密码不能与原密码一致。
- 确认新密码必须与新密码一致。
- 输入框失焦时可进行单项校验，点击“确认修改”时需进行全量校验。

### 8.5 提示文案
- 原密码为空：请输入原密码。
- 新密码为空：请输入新密码。
- 新密码长度不符合要求：新密码长度需为6-20位。
- 新密码与原密码一致：新密码不能与原密码相同。
- 确认新密码为空：请再次输入新密码。
- 两次新密码不一致：两次输入的新密码不一致。
- 修改成功：密码修改成功，请重新登录。
- 原密码错误：原密码错误，请重新输入。
- 修改失败：修改失败，请稍后重试。

### 8.6 提交流程
- 用户填写原密码、新密码、确认新密码。
- 前端完成必填、长度、一致性校验。
- 校验通过后调用修改密码接口。
- 修改成功后清除登录状态，并跳转登录页。
- 修改失败时停留当前页面并展示错误提示。

### 8.7 安全规则
- 密码输入框默认不展示明文。
- 密码内容不在页面、日志、提示文案中明文展示。
- 修改成功后需要重新登录。
- 接口提交时仅提交必要字段：原密码、新密码。

## 9. 我的寄售（权限分支）

### 9.1 未开启寄售权限用户
点击“我的寄售”后进入“寄售说明页”，展示：
- 寄售介绍说明（寄售流程、结算说明、注意事项）。
- 客服联系方式（微信号/手机号/二维码，按运营配置展示）。

### 9.2 已开启寄售权限用户
点击“我的寄售”后进入“寄售管理页”。

## 10. 寄售管理页

### 10.1 页面内容
- 抽成比例（当前用户寄售抽成比例）。
- 寄售商品入口。
- 已出售入口。
- 已结算入口。

### 10.2 页面交互
- 点击“寄售商品入口”跳转寄售商品页。
- 点击“已出售入口”跳转已出售页。
- 点击“已结算入口”跳转已结算页。

## 11. 寄售商品页

### 11.1 数据范围
- 显示当前登录用户的全部寄售商品。

### 11.2 展示与交互
- 页面右上角或列表顶部展示“添加”按钮。
- 点击“添加”按钮后打开上传商品页。
- 按时间倒序展示（最新寄售在前）。
- 支持查看商品状态标签（如上架、已出售、下架等）。
- 点击商品信息区域跳转对应的商品详情页。
- 商品状态为“已下架”时，用户可以编辑和删除该商品。
- 商品状态为“上架”“已出售”“已结算”时，不提供编辑和删除操作。
- 点击“编辑”进入商品编辑页，编辑页样式和字段与上传商品页保持一致，并回填当前商品信息。
- 点击“删除”需二次确认，确认后删除商品，删除成功后刷新寄售商品列表。

### 11.3 空状态
- 当前用户暂无寄售商品时显示空状态。
- 空状态文案：暂无寄售商品。
- 空状态页面保留“添加商品”入口，点击后进入上传商品页。

## 12. 上传商品页

### 12.1 页面目标
已开启寄售权限的用户可自行上传寄售商品，上传后的商品自动归属当前登录用户。

### 12.2 页面样式
- 页面样式与管理端“上传商品页”保持一致。
- 页面结构、表单布局、图片上传样式、字段顺序、必填标识、提交按钮样式与管理端一致。
- 用户端不展示“选择寄售用户”字段。

### 12.3 字段规则
- 商品照片，默认只显示添加按钮，最多9张，九宫格形式。
- IP，必填，独占一行。
- 角色，必填，独占一行。
- 系列，必填，独占一行。
- 价格，必填，独占一行。
- 类型，必填，下拉框。
- 自定义类型输入框，仅类型选择“自定义”时显示。
- 数量，必填。
- 质量，必填，分段选择：无暇 / 有瑕。
- 备注。
- 提交上传按钮，直接跟随表单内容显示，不使用底部遮罩层。
- 当前用户端上传不展示购买记录字段，系统默认保存为"无"。

### 12.4 业务规则
- 上传商品时系统自动绑定当前登录用户为寄售用户。
- 用户端上传商品无需选择寄售用户，也不可代其他用户上传。
- 用户端上传商品无需填写商品状态。
- 用户提交商品后，商品状态统一默认为“已下架”。
- 用户端上传商品不填写购物平台链接；购物平台链接由管理员在管理端补充后上架。
- 必填项字段名称后面显示红色“*”。
- 点击提交前需完成必填校验。
- 上传成功后返回寄售商品页并刷新列表。

## 13. 已出售页

### 13.1 数据范围
- 显示当前用户已出售商品。
- 数据仅包含当前登录用户名下的寄售商品。
- 商品状态标签为“已出售”。

### 13.2 页面样式
- 页面样式与管理端“已出售商品”页面保持一致。
- 列表结构、商品信息展示、状态标签、金额信息、时间信息与管理端已出售商品页面对齐。

### 13.3 展示与交互
- 默认按出售时间倒序展示。
- 可展示出售价格、出售时间、订单编号（以接口字段为准）。

### 13.4 空状态
- 当前用户暂无已出售商品时显示空状态。
- 空状态文案：暂无已出售商品。
- 空状态不展示商品列表，不展示分页加载。

## 14. 已结算页

### 14.1 数据范围
- 显示当前用户已结算的商品。
- 数据仅包含当前登录用户名下的寄售商品。
- 数据口径与管理端“已结算商品”页面一致。

### 14.2 页面样式
- 页面样式与管理端“已结算商品”页面保持一致。
- 列表结构、商品信息展示、结算信息、金额信息、时间信息与管理端已结算商品页面对齐。

### 14.3 展示与交互
- 显示结算金额、结算时间、结算单号/批次号（以接口字段为准）。
- 支持按时间倒序展示。

### 14.4 空状态
- 当前用户暂无已结算商品时显示空状态。
- 空状态文案：暂无已结算商品。
- 空状态不展示商品列表，不展示分页加载。

## 15. 路由与跳转关系
- 登录成功 -> IP分类页
- IP分类页 -> 点击 Banner -> 对应 IP 商品列表页
- IP分类页 -> 点击 IP 卡片 -> 对应 IP 商品列表页
- IP分类页 -> 点击搜索框 -> 搜索页
- 搜索页 -> 点击搜索结果商品 -> 商品详情页
- 搜索页 / 搜索结果入口 -> 全站搜索结果模式商品列表页
- IP商品列表页 -> 点击商品卡片 -> 商品详情页
- 我的页面 -> 修改密码 -> 修改密码页
- 我的页面 -> 我的寄售 ->
  - 未开通权限：寄售说明页（介绍说明 + 客服联系方式）
  - 已开通权限：寄售管理页
- 寄售管理页 ->
  - 寄售商品入口 -> 寄售商品页
  - 已出售入口 -> 已出售页
  - 已结算入口 -> 已结算页
- 寄售商品页 -> 添加 -> 上传商品页
- 寄售商品页 -> 点击商品信息 -> 商品详情页
- 寄售商品页 -> 已下架商品 -> 编辑 -> 商品编辑页

## 16. 状态与异常处理
- 列表类页面统一包含：加载中、空状态、加载失败三种状态。
- 网络异常时统一提示：网络异常，请稍后重试。
- 接口返回无权限时，寄售相关页面提示无权限并提供联系客服入口。
- 某个 IP 被删除或其下所有商品不再上架后，该 IP 不再出现在 IP分类页。

## 17. 验收要点（测试用）
- 用户登录后默认进入 IP分类页，而不是总商品列表页。
- 用户端不存在“默认总商品列表首页”。
- IP分类页仅展示有上架商品的 IP。
- Banner 仅展示最近上新的上架 IP，最多 3 个。
- 点击 Banner 能进入对应 IP 商品列表页。
- 点击 IP 卡片能进入对应 IP 商品列表页。
- IP分类页点击搜索框进入独立搜索页。
- 搜索页可按 IP / 角色 / 系列 / 类型搜索全部上架商品。
- 搜索页保存最近 8 条历史，并支持清空。
- 热门 IP 按搜索次数、上新时间、商品数量排序。
- IP商品列表页搜索仅搜索当前 IP 内商品。
- IP商品列表页筛选项为“角色、系列”。
- IP商品列表页不再提供 IP 筛选项。
- 商品详情不显示“寄售用户、抽成比例、已结算”。
- 商品详情需要显示商品状态。
- 商品详情需展示“在哪里买”模块，仅展示已配置的平台、店铺名称和搜索关键词，不展示商品外链 URL。
- 商品详情点击购买途径“复制”后复制店铺名或搜索关键词，并提示“店铺名已复制”。
- 商品详情需展示“联系客服”模块，点击“联系”可进入客服会话，并携带当前商品上下文。
- 修改密码页包含原密码、新密码、确认新密码三个必填项。
- 修改密码需校验必填、长度、新旧密码不同、两次新密码一致。
- 修改密码成功后清除登录状态并跳转登录页。
- 我的寄售按权限正确分流。
- 已开启权限用户可进入寄售管理，并可跳转三个子页面。
- 寄售商品页范围正确（当前用户全部寄售）。
- 寄售商品页无数据时显示“暂无寄售商品”，并保留添加商品入口。
- 寄售商品列表点击商品信息可跳转对应商品详情页。
- 寄售商品页点击“添加”可进入上传商品页。
- 用户端上传商品页与管理端上传商品页样式一致，且不显示“选择寄售用户”字段。
- 用户端上传商品页不显示商品状态字段。
- 用户端上传商品页不显示购物平台链接字段。
- 用户端上传商品页不显示购买记录字段，系统默认保存为"无"。
- 用户提交商品后默认状态为“已下架”。
- 用户端上传商品自动绑定当前登录用户为寄售用户。
- 仅“已下架”状态商品可编辑和删除，其他状态不展示编辑和删除操作。
- 已出售页范围正确（当前用户已出售商品），样式与管理端已出售商品页面一致。
- 已出售页无数据时显示“暂无已出售商品”。
- 已结算页范围正确（当前用户已结算商品），样式与管理端已结算商品页面一致。
- 已结算页无数据时显示“暂无已结算商品”。
