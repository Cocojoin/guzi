# 用户管理页面 Codex 还原开发说明

本文件用于后续小程序开发时，让 Codex 按当前视觉稿尽量 100% 还原用户管理页面。  
视觉唯一基准文件：`需求文档/用户管理页面视觉预览.html`

## 开发原则

- 以视觉稿 HTML 中的页面结构、颜色、间距、圆角、字体层级、卡片样式为准。
- 不重新设计页面，不新增无关模块，不改变现有蓝粉轻二次元管理端风格。
- 小程序实现时优先复刻视觉稿里的布局层级，再接入真实接口数据。
- 所有页面保持顶部系统级按钮安全区，页面整体向下，避免遮挡。
- 页面信息需清晰可扫读，避免装饰过量与复杂背景纹理。

## 设计 Token

| 名称 | 颜色 | 用途 |
| --- | --- | --- |
| ink | `#24364A` | 主标题、重要文本 |
| muted | `#6C7F99` | 次级说明文字 |
| soft | `#F2FAFF` | 页面浅底 |
| blue | `#B0DDFA` | 主蓝色、选中背景 |
| blue-deep | `#4B84BF` | 图标、强调蓝 |
| pink | `#FFD7E6` | 柔粉点缀 |
| pink-deep | `#E55472` | 强调/风险文字 |
| line | `rgba(217,236,248,0.92)` | 输入框/卡片边框 |
| shadow | `0 18px 38px rgba(45, 76, 120, 0.14)` | 通用阴影 |

页面背景：

- 外部背景：`radial + linear` 的蓝粉轻渐变氛围。
- 手机内背景：`linear-gradient(180deg, #FCFEFF 0%, #F2FAFF 56%, #FFF8FB 100%)`。

## 页面清单

开发页面按以下 ID 还原：

1. 用户列表：`userList`
2. 用户详情（编辑态）：`userDetail`
3. 详情已保存（只读态）：`userDetailSaved`
4. 用户商品：`userGoods`
5. 商品筛选下拉状态：`userGoodsFilters`
6. 待结算商品：`soldGoods`
7. 结算：`settlement`
8. 已结算记录：`settledList`
9. 结算详情：`settledDetail`
10. 确认弹窗：`confirmDialog`
11. 空态：`emptyUser`

## 通用布局规范

- 手机视口按视觉稿 390 x 844 适配。
- 页面内容左右留白最小 10px，卡片不贴边。
- `title` 为大号粗体，`subtitle` 为 11px 次级文字。
- 通用卡片为白底玻璃风，圆角 22px，浅蓝边框。
- 滚动区域使用分段卡片布局，卡片间距约 12px。
- 底部导航固定于用户列表页，用户 Tab 为选中态。

## 用户列表页（`userList`）

页面结构：

- 标题：用户管理
- 搜索区：左侧输入提示“搜索用户昵称 / 账号”，右侧搜索按钮。
- 筛选行：
  - 用户角色 ▼
  - 注册时间 ▼
- 用户卡片字段：
  - 头像字母圆章
  - 用户昵称
  - 账号与角色/状态说明
  - 右侧标签：抽成比例、普通、停用

当前要求：

- 删除顶部右侧两个按钮（已在视觉稿中处理）。
- 删除标题下方“管理寄售权限、抽成比例和结算记录”文案（已处理）。

## 用户详情编辑态（`userDetail`）

页面结构：

- 顶部返回 + 标题“用户详情”
- 用户头部卡：头像、昵称、角色标签、账号/联系方式
- 统计卡：寄售商品、待结算、已结算
- 用户信息卡（编辑态）：
  - 右上按钮：取消、保存
  - 字段：昵称、联系方式、抽成比例
  - 账号行已删除，不再展示
- 寄售权限卡：开关 + 说明
- 底部按钮：重置密码

## 用户详情保存态（`userDetailSaved`）

页面结构：

- 与编辑态保持同样版式。
- 用户信息卡右上为“编辑”按钮。
- 字段为只读展示：昵称、联系方式、抽成比例。
- 账号行已删除，不展示账号字段。

## 用户商品页（`userGoods`）

页面结构：

- 标题：用户昵称 + 副标题“全部寄售商品 · 数量”
- 筛选行：
  - 所有状态 ▼
  - 类型 ▼
  - 时间 ▼
- 需显示下拉项列表（展开态）：
  - 所有状态：全部状态、已上架、已下架、已售出
  - 类型：全部类型、镭射票、拍立得、吧唧
  - 时间：最近全部、近 7 天、近 30 天、更早
- 商品卡片字段：
  - 缩略图
  - 商品名
  - IP
  - 数量
  - 类型
  - 价格
  - 状态标签

## 商品筛选下拉状态页（`userGoodsFilters`）

用途：

- 单独展示用户商品筛选下拉“已展开+选中态”。
- 默认选中示例：
  - 状态：已上架
  - 类型：镭射票
  - 时间：近 30 天

## 待结算页（`soldGoods`）

页面结构：

- 标题：待结算商品
- 副标题：用户名 + 已售出件数
- 商品卡片字段：
  - 商品名
  - 售出数量
  - 售出抽成
  - 金额
  - 选中点
- 底部固定栏：
  - 已选数量
  - 合计寄售价
  - 结算按钮

## 结算页（`settlement`）

页面结构：

- 顶部用户信息卡
- 本次结算商品列表
- 金额汇总卡：
  - 商品总额
  - 平台抽成
  - 应付寄售用户
- 商品实际收入输入区
- 结算凭证上传区（网格）
- 提交结算按钮

## 已结算记录页（`settledList`）

页面结构：

- 记录卡列表，字段包括：
  - 日期
  - 件数
  - 商品总额
  - 抽成
  - 应付金额

## 结算详情页（`settledDetail`）

页面结构：

- 标题 + 日期 + 件数
- 商品明细列表（金额 x 数量 + 状态）
- 金额汇总卡：
  - 商品总额
  - 抽成金额
  - 商品实际收入
  - 已付寄售用户
- 结算凭证图片区

## 确认弹窗页（`confirmDialog`）

页面结构：

- 背景页保留当前信息卡。
- 中央弹窗包含：
  - 标题：关闭寄售权限？
  - 说明文案
  - 按钮：取消 / 确认

## 空态页（`emptyUser`）

页面结构：

- 标题：用户管理
- 副标题：搜索结果为空状态
- 空态图形占位
- 文案：
  - 未找到匹配结果
  - 可尝试更换昵称、账号或筛选条件

## 下拉组件规范

- 下拉按钮：浅色文字 + `▼` 指示。
- 展开容器：
  - 圆角 15px 左右
  - 白底或极浅蓝底
  - 浅蓝边框
  - 轻阴影
- 选中项：浅蓝底 + 右侧对勾 `✓`
- 选项文字必须单行显示，不截断重要字段。

## 状态与标签规范

- 抽成比例：浅蓝胶囊标签。
- 普通用户：浅紫胶囊标签。
- 停用：灰蓝胶囊标签。
- 商品状态：
  - 已上架：浅蓝状态
  - 已下架：灰蓝状态
  - 已售出：沿用状态标签视觉体系

## 关键交互路径

1. 用户列表 -> 用户详情（编辑态）
2. 用户详情保存 -> 详情已保存（只读态）
3. 用户详情点击编辑 -> 回到编辑态
4. 用户详情 -> 用户商品
5. 用户商品筛选 -> 商品筛选下拉状态页（用于表现展开态）
6. 待结算 -> 结算 -> 提交结算
7. 已结算记录 -> 结算详情
8. 风险操作 -> 确认弹窗

## 开发验收清单

- 顶部系统区域无遮挡，页面内容整体下移。
- 用户详情编辑态与保存态字段正确，均不显示“账号”行。
- 用户商品页三类筛选均有下拉列表项。
- 新增“商品筛选下拉状态”页面可切换预览。
- 用户列表右上角只保留搜索按钮，不再有其它操作按钮。
- 卡片、边框、圆角、阴影与视觉稿一致。
- 底部导航样式与选中态表现一致。
- 结算流程页字段完整、层级清晰。
- 空态页文案与布局完整。
