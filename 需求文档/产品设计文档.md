# 谷圈星社微信小程序产品设计文档

## 1. 项目概述

### 1.1 产品定位

本小程序用于管理用户寄售的二次元谷子、明星小卡等商品，支持寄售用户提交商品，管理员审核和管理商品，普通用户浏览寄售商品。

### 1.2 用户角色

| 角色 | 说明 | 核心能力 |
| --- | --- | --- |
| 管理员 | 平台运营和商品管理人员 | 管理账号、审核寄售商品、上下架商品、维护商品信息 |
| 寄售用户 | 有商品需要寄售的用户 | 注册登录、提交寄售商品、查看自己的寄售记录 |
| 普通用户 | 浏览商品的用户 | 注册登录、浏览寄售商品、查看商品详情 |

### 1.3 当前设计范围

截至 2026-06-05，当前项目已覆盖以下范围：

| 范围 | 已有文档 / 页面 |
| --- | --- |
| 登录注册 | 登录页、注册页、填写资料页 |
| 用户端商品浏览 | 商品列表、商品详情、搜索与筛选 |
| 用户端我的与寄售 | 我的页、修改密码、寄售说明、寄售管理、寄售商品、上传、已出售、已结算 |
| 管理端商品管理 | 商品列表、详情、上传、编辑、批量状态、批量售出、批量上架链接、批量上传 |
| 管理端统计 | 统计首页、收入明细、材料费用、物流费用、统计汇总 |
| 管理端用户管理 | 用户列表、用户详情、用户商品、待结算、结算、已结算 |
| 管理端设置 | 操作日志、数据备份、数据导出 |

后续功能迭代以各模块拆分需求文档为准。

## 2. 登录注册功能

### 2.1 功能目标

登录注册功能用于建立小程序的基础账号体系。所有用户需要通过账号和密码登录后，才能进入对应角色的功能页面。

本小程序暂不使用微信手机号授权、验证码登录或第三方登录，统一采用账号密码方式。

### 2.2 适用角色

| 角色 | 是否需要登录 | 是否可注册 | 说明 |
| --- | --- | --- | --- |
| 管理员 | 是 | 否 | 管理员使用系统预置专属账号，不开放前台注册 |
| 寄售用户 | 是 | 是 | 注册后可提交寄售商品 |
| 普通用户 | 是 | 是 | 注册后可浏览商品 |

### 2.3 管理员专属账号

管理员账号为系统预置账号，用于进入管理员商品管理页面。

| 字段 | 内容 |
| --- | --- |
| 管理员账号 | admin |
| 管理员密码 | cc19980905 |

说明：管理员账号属于预置账号，不受普通用户注册账号“6-12位数字和字母组合”的格式限制。普通用户和寄售用户注册时仍需遵守账号格式规则。

### 2.4 页面清单

| 页面 | 页面用途 |
| --- | --- |
| 登录页 | 用户输入账号和密码，完成登录 |
| 注册页 | 新用户填写账号、密码、确认密码，完成注册 |
| 填写资料页 | 普通用户注册成功后填写头像和昵称 |
| 忘记密码页 | 当前版本不需要 |

### 2.5 登录页设计

#### 2.5.1 页面元素

| 元素 | 类型 | 说明 |
| --- | --- | --- |
| 小程序名称 | 文本 | 展示产品名称，例如“谷圈星社” |
| 账号输入框 | 输入框 | 用户输入账号 |
| 密码输入框 | 密码输入框 | 用户输入密码，默认隐藏输入内容 |
| 登录按钮 | 主按钮 | 点击后校验账号和密码 |
| 注册入口 | 文本按钮 | 跳转到注册页 |

#### 2.5.2 输入规则

| 字段 | 规则 |
| --- | --- |
| 账号 | 普通用户和寄售用户账号为6-12位，由数字和字母组合构成，必须同时包含数字和字母；管理员账号为预置账号admin |
| 密码 | 6-12位，由数字或字母构成 |

说明：密码不允许输入特殊符号。

#### 2.5.3 登录流程

1. 用户进入登录页。
2. 用户输入账号和密码。
3. 点击“登录”按钮。
4. 系统校验账号和密码格式。
5. 格式不正确时，在对应输入框下方展示错误提示。
6. 格式正确后，系统校验账号是否存在、密码是否正确。
7. 登录成功后，根据用户角色进入对应首页。

#### 2.5.4 登录成功后的跳转规则

| 用户角色 | 登录后跳转 |
| --- | --- |
| 管理员 | 管理页面 |
| 寄售用户 | 寄售用户首页或我的寄售页 |
| 普通用户 | 商品浏览首页 |

#### 2.5.5 异常提示

| 场景 | 提示文案 |
| --- | --- |
| 账号为空 | 请输入账号 |
| 密码为空 | 请输入密码 |
| 账号格式错误 | 账号需为6-12位数字和字母组合 |
| 密码格式错误 | 密码需为6-12位数字或字母 |
| 账号不存在 | 账号不存在，请先注册 |
| 密码错误 | 密码错误，请重新输入 |
| 账号被禁用 | 账号已被禁用，请联系管理员 |

### 2.6 注册页设计

#### 2.6.1 页面元素

| 元素 | 类型 | 说明 |
| --- | --- | --- |
| 页面标题 | 文本 | 展示“注册账号” |
| 账号输入框 | 输入框 | 新用户填写账号 |
| 密码输入框 | 密码输入框 | 新用户填写密码 |
| 再次输入密码框 | 密码输入框 | 用于确认两次密码一致 |
| 注册按钮 | 主按钮 | 点击后提交注册 |
| 返回登录入口 | 文本按钮 | 返回登录页 |

#### 2.6.2 输入规则

| 字段 | 规则 |
| --- | --- |
| 账号 | 6-12位，由数字和字母组合构成，必须同时包含数字和字母 |
| 密码 | 6-12位，由数字或字母构成 |
| 再次输入密码 | 必须与密码完全一致 |

#### 2.6.3 注册流程

1. 用户进入注册页。
2. 用户填写账号、密码、再次输入密码。
3. 点击“注册”按钮。
4. 系统校验账号格式、密码格式、两次密码是否一致。
5. 如果校验失败，在对应字段下方展示错误提示。
6. 如果校验通过，系统检查账号是否已被注册。
7. 账号未被注册时，创建普通用户账号。
8. 注册成功后，进入填写资料页。

### 2.7 填写资料页设计

#### 2.7.1 页面触发时机

普通用户注册成功后，系统自动进入填写资料页。用户需要完成资料填写后，才能进入商品浏览首页。

管理员不进入填写资料页。管理员登录成功后直接进入管理页面。

#### 2.7.2 页面元素

| 元素 | 类型 | 说明 |
| --- | --- | --- |
| 页面标题 | 文本 | 展示“填写资料” |
| 头像 | 图片上传/默认头像 | 用户可本地上传头像；未上传时默认展示昵称首字头像 |
| 昵称输入框 | 输入框 | 默认显示当前账号，用户可修改为昵称 |
| 完成按钮 | 主按钮 | 保存资料并进入商品浏览首页 |

#### 2.7.3 输入规则

| 字段 | 是否必填 | 规则 |
| --- | --- | --- |
| 头像 | 否 | 支持本地上传，上传图片大小不超过5M；用户不上传时默认显示昵称首字头像；昵称未修改前默认使用账号首字 |
| 昵称 | 是 | 支持中文、英文、数字，长度不超过12个字 |

#### 2.7.4 提交流程

1. 用户注册成功后进入填写资料页。
2. 昵称输入框默认回填当前账号。
3. 系统默认展示昵称首字头像；若用户尚未修改昵称，则展示账号首字头像。
4. 用户可点击头像区域，从本地上传头像。
5. 用户填写或修改昵称时，默认头像需同步更新为最新昵称首字。
6. 点击“完成”按钮。
7. 系统校验昵称是否为空、格式和长度是否符合要求。
8. 校验通过后保存头像和昵称。
9. 保存成功后进入商品浏览首页。

#### 2.7.5 异常提示

| 场景 | 提示文案 |
| --- | --- |
| 昵称为空 | 请输入昵称 |
| 昵称格式错误 | 昵称仅支持中文、英文和数字 |
| 昵称超过长度 | 昵称不能超过12个字 |
| 头像超过大小限制 | 头像图片不能超过5M |
| 头像上传失败 | 头像上传失败，请重新选择 |
| 资料保存失败 | 保存失败，请稍后重试 |

### 2.8 注册账号角色规则

前台注册默认创建普通用户账号。

寄售用户身份不在注册页直接选择，建议在后续“申请寄售”或“上传寄售商品”流程中完成身份升级。这样可以避免普通用户误选角色，也方便管理员对寄售权限进行管理。

如业务希望注册时直接选择角色，也可以在注册页增加“账号类型”选择项：

| 账号类型 | 说明 |
| --- | --- |
| 普通用户 | 默认类型，可浏览商品 |
| 寄售用户 | 可提交寄售商品，可能需要管理员审核 |

当前确定方案：注册页不选择角色，默认普通用户。

### 2.10 用户数据模型

注册、登录、资料填写依赖 `users` 表存储用户账号数据，后续商品寄售、代理分佣、后台管理也基于该表扩展。

#### 2.10.1 `users` 表字段

| 字段名 | 类型 | 是否必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| account | String | 是 | `abc123` | 登录账号，唯一 |
| nickname | String | 否 | `小星` | 用户昵称，普通用户注册后补填 |
| password | String | 是 | `abc123` | 登录密码，正式开发建议加密存储 |
| role | String | 是 | `normal_user` | 用户角色 |
| isAgentEnabled | Boolean | 是 | `false` | 是否开启代理权限 |
| platformRate | Number | 否 | `0.15` | 平台对寄售商品成交的抽成比例 |
| commissionRate | Number | 否 | `0.1` | 开启代理权限后可获得的佣金比例 |
| contactWechat | String | 否 | `star_shop01` | 微信联系方式 |
| contactMobile | String | 否 | `13800000000` | 手机号联系方式 |
| avatarUrl | String | 否 | `cloud://env/avatar.png` | 头像地址 |
| status | String | 是 | `active` | 账号状态，`active` / `disabled` |
| createdAt | Date | 是 | `2026-05-19T10:00:00Z` | 创建时间 |
| updatedAt | Date | 是 | `2026-05-19T10:00:00Z` | 更新时间 |

#### 2.10.2 `role` 枚举值

| 枚举值 | 中文角色 | 说明 |
| --- | --- | --- |
| admin | 管理员 | 系统后台管理账号 |
| consignment_user | 寄售用户 | 可提交寄售商品 |
| normal_user | 普通用户 | 前台默认注册角色 |

#### 2.10.3 字段规则

1. `account` 必须唯一。
2. `nickname` 当前版本可为空，但普通用户注册完成后必须补填。
3. `password` 当前 PRD 按账号密码模型定义，开发时应改为加密存储。
4. `role` 默认值为 `normal_user`。
5. `isAgentEnabled` 默认值为 `false`，由管理员为普通用户或寄售用户手动开启。
6. `platformRate` 对寄售用户生效，其他角色可为空。
7. `commissionRate` 对已开启代理权限的用户生效，未开启代理权限时可为空。
8. `contactWechat` 和 `contactMobile` 至少保留一个可填写入口，当前版本允许均为空。
9. `status` 默认为 `active`，若为 `disabled` 则禁止登录。

### 2.11 注册、登录与数据表关系

#### 2.11.1 注册写入规则

1. 用户在注册页提交账号、密码、确认密码。
2. 系统根据 `account` 查询 `users` 表，判断是否已存在。
3. 若账号不存在，则写入一条新用户记录。
4. 默认写入值如下：
   - `role = normal_user`
   - `isAgentEnabled = false`
   - `status = active`
   - `nickname = ""`
   - `avatarUrl = ""`
   - `platformRate = null`
   - `commissionRate = null`
   - `contactWechat = ""`
   - `contactMobile = ""`
5. 注册成功后进入填写资料页，补充 `nickname` 和 `avatarUrl`。

#### 2.11.2 登录查询规则

1. 用户输入账号和密码后，系统按 `account` 查询 `users` 表。
2. 若未查询到用户，提示“账号不存在，请先注册”。
3. 若查询到用户但 `status = disabled`，提示“账号已被禁用，请联系管理员”。
4. 若密码不匹配，提示“密码错误，请重新输入”。
5. 若登录成功，则根据 `role` 进入不同首页。

#### 2.11.3 代理权限规则

1. 代理不是独立角色，而是一项附加权限。
2. 管理员可为 `normal_user` 或 `consignment_user` 开启代理权限。
3. 开启代理权限后：
   - `isAgentEnabled = true`
   - 可配置 `commissionRate`
4. 关闭代理权限后：
   - `isAgentEnabled = false`
   - `commissionRate` 可保留历史值或清空，当前建议由后台决定

#### 2.11.4 资料页更新规则

1. 普通用户注册成功后进入填写资料页。
2. 用户提交资料后，更新 `nickname`、`avatarUrl`、`updatedAt`。
3. 若头像未上传，则允许保留空值或由系统生成默认头像。

### 2.12 交互细节

1. 密码输入框默认隐藏内容，可提供显示/隐藏密码按钮。
2. 点击登录或注册按钮后，如果正在提交，按钮进入加载状态，避免重复点击。
3. 输入框失焦后可进行格式校验，提交时再次进行完整校验。
4. 错误提示显示在输入框下方，用户修改内容后可自动清除对应错误。
5. 登录成功后记录登录状态，用户下次打开小程序时可自动进入对应首页。
6. 退出登录后返回登录页。
7. 当前版本不提供忘记密码入口。
8. 普通用户注册后必须先完成昵称填写；头像未上传时自动使用默认头像。
