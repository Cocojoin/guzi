# 二、商品管理

商品管理用于管理平台内寄售商品的信息、状态、上传、编辑和批量处理操作。

## 0.0 当前开发同步说明

截至 2026-06-16，商品管理需求以当前小程序已实现能力为准：

| 功能 | 当前开发现状 | 当前需求口径 |
|---|---|---|
| 商品列表 | 已接入云数据库商品集合；支持搜索、寄售用户筛选、状态筛选、多选、批量删除、批量状态入口 | 保持当前交互；筛选项二期再补商品类型 / 质量 |
| 单个上传 | 已支持图片、寄售用户、基础字段、状态、备注；当前不包含购物平台链接录入 | 保持当前实现 |
| 批量状态 | 已独立为 `batch-status` 页面；可选择上架 / 下架 / 标记售出；标记售出进入 `batch-sold` 页面填写数量和售价 | 文档已同步，批量售出仅允许对仍有剩余可售数量的商品执行 |
| 批量上架链接 | 已独立为 `batch-links` 页面；每件商品固定展示拼多多、淘宝/天猫、小红书 3 个链接输入框 | 保持当前形态 |
| 批量上传 | 当前代码支持 `.csv/.xlsx`；解析模板列：`IP/角色/系列/类型/数量/价格/质量/状态/图片/备注`；图片通过"图片编号"匹配已选图片顺序 | 保持当前实现 |
| 操作日志 | 商品新增、批量删除、批量状态、批量售出、批量链接上架已写入操作日志 | 保持并补齐失败日志 |

## 0. 模块目标与字段口径

### 0.1 功能目标

商品管理模块需支持管理员完成以下闭环：

| 目标 | 说明 |
|---|---|
| 商品建档 | 录入寄售商品基础信息、图片、价格、数量、归属用户、质量、购买记录、备注 |
| 上下架管理 | 通过单件 / 批量方式控制商品是否对外展示；批量上架链接页提交时需补充合法购物平台链接 |
| 售出记录 | 支持单件或多数量商品按数量标记售出，剩余可售数量自动影响展示状态 |
| 信息维护 | 商品未结算前可按权限编辑信息；已结算商品只读 |
| 批量效率 | 支持批量删除、批量上下架、批量标记售出、批量上传和批量上架链接填写，减少重复录入 |

### 0.2 商品字段模型

| 字段 | 说明 | 必填 | 生成 / 维护规则 |
|---|---|---|---|
| 商品编号 `id` | 商品唯一标识，展示为 `#A00128` | 是 | 系统生成，不允许手动修改；新增时按现有最大编号递增 |
| 图片 `images` | 商品实拍图 | 是 | 最多 9 张，首图为封面；当前版本若未接入图片存储，可展示默认占位图 |
| 寄售用户 `owner` | 商品归属的寄售用户 | 是 | 仅可选择有寄售权限的用户；更换归属需二次确认 |
| IP `ip` | 作品 / 团体 / IP 名称 | 是 | 1-12 字 |
| 系列 `series` | 商品系列、活动或批次 | 是 | 1-12 字 |
| 角色 `role` | 角色 / 成员 / 人物名称 | 是 | 1-12 字 |
| 类型 `type` | 商品类型 | 是 | 小卡 / 吧唧 / 镭射票 / 自定义 |
| 自定义类型 `customType` | 自定义类型名称 | 条件必填 | 类型为"自定义"时必填，1-12 字 |
| 总数量 `totalQuantity` | 该商品总件数 | 是 | 正整数，不得小于 `soldCount + settledCount` |
| 已售出数量 `soldCount` | 已售出未结算数量 | 是 | 默认 0；详情页售出操作累加 |
| 已结算数量 `settledCount` | 已完成结算数量 | 是 | 默认 0；由结算页累加 |
| 价格 `price` | 单件售卖价格 | 是 | 大于 0，最多 2 位小数 |
| 质量 `quality` | 商品品相 | 是 | 有瑕 / 无暇 |
| 购买记录 `purchaseRecord` | 是否保留购买记录 | 是 | 有 / 无 |
| 基础状态 `status / base_status` | 人工维护的上下架状态 | 是 | 已下架 / 已上架；展示状态按 §4 计算 |
| 购物平台链接 `links` | 商品在外部平台的购买链接 | 条件必填 | 批量上架链接页提交时至少填写 1 条合法链接；最多 3 条 |
| 购买途径 `purchaseChannels` | 商品可展示的购买平台与内部链接 | 条件必填 | 商品上架后至少 1 个有效购买途径；店铺名称与搜索关键词从设置模块统一读取；用户端不展示商品链接 |
| 备注 `remark` | 补充说明 | 否 | 0-200 字 |
| 创建时间 `createdAt` | 商品建档时间 | 是 | 系统生成 |
| 更新时间 `updatedAt` | 最近保存 / 状态变更时间 | 是 | 系统更新 |

### 0.3 平台链接统一规则

| 平台 | 可接受域名示例 | 说明 |
|---|---|---|
| 拼多多 | `yangkeduo.com`、`pinduoduo.com` | 含移动端链接 |
| 淘宝 / 天猫 | `taobao.com`、`tmall.com` | 淘宝与天猫归为同一平台，不能重复 |
| 小红书 | `xiaohongshu.com`、`xhslink.com` | 含短链 |

🟩 统一校验：
- 链接必须以 `http://` 或 `https://` 开头。
- 同一商品内，同一平台最多保留 1 条链接。
- 批量上架链接页固定展示 3 个平台输入框；编辑等链接维护场景允许保存 1-3 条平台链接。
- 保存时只提交合法链接；空链接不保存。
- 批量上架链接页提交时，如果某商品没有合法链接，必须阻断；编辑已上架商品时如清空全部合法链接，必须阻断。

### 0.3.1 购买途径统一规则

购买途径用于用户端商品详情页“在哪里买”模块展示，和内部购物平台链接分开管理。店铺名称与搜索关键词由设置模块统一维护，商品侧只维护该商品支持哪些平台以及内部购买链接。

| 字段 | 说明 | 规则 |
|---|---|---|
| 平台 | 淘宝/天猫、拼多多、小红书 | 同一商品同一平台最多 1 条 |
| 是否展示 | 控制该商品是否在用户端展示该平台 | 平台需先在设置模块启用；关闭后用户端不显示该平台 |
| 内部购买链接 | 客服或管理端使用的商品链接 | 可选；不在用户端展示；填写时按 §0.3 校验 |

🟩 展示与安全口径：
- 用户端仅展示平台、全局店铺名称、全局搜索关键词，不展示内部购买链接。
- 用户端提供“复制店铺名 / 搜索关键词”和“联系客服”能力，不提供外链跳转。
- 客服可基于内部购买链接向用户发送对应商品链接。
- 商品为已上架时，至少需要 1 个启用展示的购买途径。
- 批量上架链接页填写的合法平台链接可作为内部购买链接；店铺名称和搜索关键词从设置模块“店铺信息设置”读取。

### 0.4 权限与只读口径

| 场景 | 规则 |
|---|---|
| 管理员 | 可访问商品管理全部功能 |
| 非管理员误入 | 跳转登录页或无权限页，Toast "暂无管理权限" |
| 已结算商品 | 列表可查看，详情只读，不允许编辑、删除、修改状态、批量选择 |
| 已售出商品 | 可查看、可编辑非归属核心信息；不可删除；状态仅支持详情页退款回退为已上架 |

## 1. 商品列表

### 1.2 🟨 交互规则

**查看**
- 点击商品行 → 进入商品详情页。
- 下拉刷新、上拉加载更多遵循通用规则。
- 搜索遵循通用规则，支持按角色 / IP / 系列 / 寄售用户昵称 / 商品编号模糊匹配。
- 搜索关键字去除首尾空格；英文大小写不敏感；商品编号输入 `A00128` 或 `#A00128` 均可命中。
- 筛选遵循通用规则；搜索与筛选同时存在时取交集。

**多选模式进入与退出**
- 顶部右侧点击"多选" → 进入多选模式；标题切换为"已选 N 项"；右上角变为"取消"。
- 多选模式下页面顶部增加"全选 / 取消全选"切换；底部固定栏出现批量操作（删除 / 修改状态）。
- 点击"取消"或物理返回键 → 退出多选模式，已勾选全部清空。

**全选规则**
- 全选只勾选当前筛选结果内的**可操作商品**（已下架 + 已上架）。
- 已售出 / 已结算商品**自动排除**，不被勾选。
- 当前筛选结果无任何可选商品时，"全选"按钮置灰、不可点击。
- 已勾选商品在翻页、搜索或筛选后仍保留；如果商品因状态变更变为不可操作，则自动从勾选中移除。

**批量操作**
- 至少选中 1 件可操作商品后，底部操作按钮才变为常态可点击；否则置灰。
- 批量操作提交前需基于最新数据再次校验，避免列表数据过期导致误操作。

### 1.2.1 排序与刷新

| 项 | 规则 |
|---|---|
| 默认排序 | 按创建时间倒序；同一创建时间按商品编号倒序 |
| 手动刷新 | 下拉刷新后清空分页游标，保留搜索与筛选条件 |
| 刷新后勾选 | 多选模式下刷新成功后，保留仍存在且仍可操作的已选商品 |
| 删除后刷新 | 删除成功后回到第一页，并清空选择 |
| 状态修改后刷新 | 状态修改成功后刷新当前筛选结果；若商品不再满足筛选条件则从列表移除 |

### 1.3 批量操作 · 批量删除

| 步骤 | 行为 |
|---|---|
| 1. 点击"批量删除" | Dialog 二次确认："确认删除所选商品吗？删除后不可恢复" |
| 2. 点击"确认" | 按钮显示加载态，请求服务端 |
| 3. 成功 | 退出多选模式，刷新列表，Toast "删除成功" |
| 4. 失败 | Toast "删除失败，请稍后重试"；保持多选模式与勾选状态 |

🟧 失败兜底：
- 未选择商品点击批量删除 → Toast "请先选择商品"
- 选中包含不可操作商品（理论上不会发生） → Toast "已售出或已结算商品不支持该操作"

### 1.4 批量操作 · 批量修改状态

批量修改状态支持以下目标状态：

| 当前状态 → 目标状态 | 说明 |
|---|---|
| 已下架 → 已上架 | 触发"购物平台链接填写"流程（见 §1.5） |
| 已上架 → 已下架 | 无需额外信息 |
| 已上架 / 已下架 → 已售出 | 进入"批量标记售出"流程（见 §1.4.1），需逐件填写售出数量 |

🟩 业务规则：

- **批量上架 / 下架**只允许在"已下架"与"已上架"之间互转。
- **批量标记售出**通过独立页面执行，不在 ActionSheet 中直接完成；需逐件填写售出数量，不能一键整批售出。
- **已售出 → 已结算 不支持批量**。该流转通过**结算页面**执行。
- **已结算 不可流转**。任何方向的批量操作都自动跳过已结算商品。
- 若所选商品中存在已结算商品，整批操作**全部阻断**，提示用户重新选择。

🟨 交互流程：

1. 点击底部"修改状态" → 进入批量状态页，目标状态展示：**全部上架 / 全部下架 / 标记售出**。
2. 系统先校验所选商品状态：
   - 全部可流转且与目标状态不同 → 进入下一步。
   - 包含已结算 / 与目标状态相同 / 无剩余可售数量的商品 → Toast "请仅选择可流转的商品后重试"，**阻断本次操作**。
3. 选择目标状态后的处理：
   - **全部下架** → Dialog 确认 "确认将所选 N 件商品下架吗？" → 确认后请求 → Toast "状态修改成功"。
   - **全部上架** → 跳转"购物平台链接填写"页（见 §1.5）。
   - **标记售出** → 跳转"批量标记售出"页（见 §1.4.1）。

🟧 提示：

| 场景 | 文案 | 形式 |
|---|---|---|
| 含已售出 / 已结算商品 | 请仅选择可流转的商品后重试 | Toast |
| 含已售出商品 | 已售出商品请到详情页单独修改状态 | Toast |
| 包含与目标状态相同的商品 | 部分商品已是该状态，请重新选择 | Toast |
| 已下架批量上架前 | 跳转购物平台链接填写页（见 §1.5） | — |
| 已上架批量下架前 | 确认将所选 N 件商品下架吗？ | Dialog |
| 批量标记售出前 | 跳转批量标记售出页 | — |
| 修改成功 | 状态修改成功 | Toast |

#### 1.4.1 批量标记售出

🟦 显示规则：
- 进入独立页面，标题"批量标记售出"。
- 列表展示所选商品：商品标题、剩余可售数量、售出数量输入框、成交价输入框。
- 售出数量默认为空；成交价默认带入商品标价，可修改。
- 页面底部主按钮"确认标记售出"。

🟩 业务规则：
- 至少 1 件商品填写售出数量后才能提交。
- 单件商品售出数量必须为正整数，且不得超过该商品剩余可售数量。
- 成交价必须大于 0；若为空或非法，默认使用商品标价。
- 提交成功后累加 `soldCount`，并按寄售用户当前抽成生成售出批次 `soldBatches`，用于后续结算锁定抽成。
- 批量售出成功后写入操作日志，记录商品编号与售出数量。

🟧 提示：

| 场景 | 文案 | 形式 |
|---|---|---|
| 未填写任何售出数量 | 请至少填写 1 个商品的售出数量 | Toast |
| 数量超过剩余可售 | 售出数量不能超过剩余可售数量 | 就近 |
| 提交成功 | 已标记售出 | Toast |
| 提交失败 | 提交失败，请重试 | Toast |

### 1.5 批量修改为"已上架" · 购物平台链接填写

🟦 显示规则：

- 进入独立页面，标题"批量上架（共 N 件）"。
- 列表展示所选商品（缩略图 + 标题 + 编号），**每件商品下方展示"购物平台链接组"**。
- 每件商品固定展示 3 行链接输入框：拼多多链接、淘宝/天猫链接、小红书链接。
- 不提供新增 / 删除平台行；空链接不保存。
- 顶部说明文案："商品修改为已上架前，至少需填写 1 条购物平台链接。支持拼多多、淘宝（含天猫）、小红书。"
- 底部主按钮"提交批量上架"。

🟩 业务规则（手动填写）：

- 每件商品必须**单独维护自己的购物平台链接**，不支持"统一一个链接应用到全部商品"。
- 每件商品在提交上架时，**至少存在 1 条合法链接**；存在多平台链接时可一并保存。
- 支持的链接域名白名单：拼多多 / 淘宝（含天猫）/ 小红书。链接需以 `http(s)://` 开头。
- 同一商品内，平台值不可重复；重复时阻断提交。
- 校验：所选商品**全部**满足"至少 1 条合法链接"后，"提交批量上架"按钮方可点击。

#### 1.5.1 手动填写交互

- 输入框失焦时按当前平台做单字段校验。
- 链接非白名单 / 格式错 → 就近提示 "请填写拼多多 / 淘宝 / 小红书的合法链接"。
- 同一商品未填写任何合法链接 / 平台重复 → 提交按钮置灰。
- 提交成功 → 退出当前页，刷新商品列表，Toast "批量上架成功"。

🟧 提示：

| 场景 | 文案 | 形式 |
|---|---|---|
| 单字段为空 | 请填写链接 | 就近 |
| 单字段格式错 | 链接格式不正确，请检查 | 就近 |
| 单字段域名不在白名单 | 仅支持拼多多 / 淘宝 / 小红书链接 | 就近 |
| 同商品平台重复 | 同一商品不能重复填写相同平台链接 | 就近 |
| 提交时仍有未填项 | 请为每件商品至少填写 1 条合法链接后再上架 | Toast |
| 提交成功 | 批量上架成功 | Toast |

### 1.6 🟧 提示规则汇总

## 3. 批量上传

### 3.1 录入流程

1. 点击"批量上传"进入流程页。
2. 上传表格文件（.csv / .xlsx 单文件；.xls 暂不支持，需另存为 .xlsx）。
3. 上传商品图片（多张图片，系统按选择顺序形成图片编号）。
4. 选择本次批量上传所归属的**寄售用户**（单选下拉）。
5. 点击"提交"。

### 3.2 🟦 显示规则

| 区块 | 显示 |
|---|---|
| 表格上传 | 单文件区域，已选文件显示文件名 + ✕ 移除按钮；提供"下载模板"链接，当前生成 CSV 模板 |
| 图片上传 | 多文件区域，缩略图网格；右上角"✕"逐张移除 |
| 寄售用户 | 下拉选择，与单个上传一致 |
| 提交按钮 | 灰态条件：表格 / 图片 / 用户 任一为空；常态：三者齐全 |

### 3.2.1 导入模板设计

批量上传当前使用单表模板，首行为字段表头，第二行起填写商品。页面点击"下载模板"时生成 CSV 文件。

| 列 | 字段 | 必填 | 规则 |
|---|---|---|---|
| A | IP | 是 | 1-12 字 |
| B | 角色 | 是 | 1-12 字 |
| C | 系列 | 是 | 1-12 字 |
| D | 类型 | 是 | 小卡 / 吧唧 / 镭射票 / 自定义等文本 |
| E | 数量 | 是 | 正整数，大于 0 |
| F | 价格 | 是 | 大于 0 |
| G | 质量 | 是 | 有瑕 / 无暇；非"有瑕"默认按"无暇"处理 |
| H | 状态 | 是 | 已上架 / 已下架；非"已上架"默认按"已下架"处理 |
| I | 图片 | 是 | 填写图片编号，如 `1` 或 `1.jpeg`，对应图片上传区第 1 张图片 |
| J | 备注 | 否 | 0-200 字；为空时默认"由批量上传创建。" |

🟩 模板交互与解析规则：
- 支持 `.csv` 与 `.xlsx`；`.xls` 当前提示"暂不支持 .xls，请另存为 .xlsx"。
- 系统按表头名称解析列，缺少任一必需列时提示"模板列不匹配，请重新下载最新模板"。
- 批量上传不解析商品编号，商品编号由系统按现有最大编号递增生成。
- 批量上传当前不解析购物平台链接。
- 批量上传的购买记录当前统一保存为"无"。

### 3.2.2 图片与表格对应规则

批量上传时，图片先在页面中选择，系统按选择顺序为图片编号；表格的 `图片` 列填写编号来匹配图片。

| 图片列填写 | 匹配规则 |
|---|---|
| `1` | 匹配本次图片上传区第 1 张图片 |
| `2` | 匹配本次图片上传区第 2 张图片 |
| `1.jpeg` / `1.jpg` / `1.png` / `1.webp` | 取文件名前的数字 `1`，匹配第 1 张图片 |

🟩 规则：
- 图片编号从 1 开始，不是从 0 开始。
- 当前每个 Excel / CSV 行只匹配 1 张图片。
- 图片列为空、非数字、编号小于 1、编号大于已上传图片数量时，该行不导入，并提示图片编号无效。
- 图片文件上传到云存储后，商品记录保存匹配到的单张图片路径。

### 3.3 🟨 交互规则

- 表格字段必须满足模板要求，至少包含 `IP` / `角色` / `系列` / `类型` / `数量` / `价格` / `质量` / `状态` / `图片`。

- 商品编号由系统递增生成，不读取表格中的商品编号。

## 4. 多数量商品状态规则

当商品**总数量 > 1** 时，商品的显示状态由**剩余可售数量**决定，**不再单独维护一个"部分售出"状态**。

### 4.1 状态显示规则

商品在**列表 / 详情**中显示的状态标签按以下顺序判定：

```text
if 已结算数量 == 总数量:
    显示"已结算"
elif (已售出数量 + 已结算数量) == 总数量:
    显示"已售出"        ← 数量为 0（即剩余可售为 0）才显示
else:
    显示 base_status（已下架 / 已上架）
```

| 总数量 | 已售出 | 已结算 | 剩余可售 | base_status | 显示状态 |
|---|---|---|---|---|---|
| 5 | 0 | 0 | 5 | 已上架 | **已上架** |
| 5 | 2 | 0 | 3 | 已上架 | **已上架**（仍可售） |
| 5 | 2 | 0 | 3 | 已下架 | **已下架** |
| 5 | 5 | 0 | 0 | 已上架 | **已售出** |
| 5 | 3 | 2 | 0 | 已上架 | **已售出** |
| 5 | 0 | 5 | 0 | — | **已结算** |

> 关键点：多数量商品在被部分售出后，**只要还有剩余可售**，状态仍跟随 base_status。**只有剩余可售为 0 时才显示"已售出"**。这样商品在列表中的显示既反映"是否还能买到"，又反映管理操作的最终状态。

🟩 补充说明：
- 此处"数量为 0"专指 **剩余可售数量为 0**，不是总数量字段被清空。
- 上述判定同时适用于**商品列表**与**商品详情页**，两处状态展示口径必须完全一致。
- `base_status` 需要持久化保存为最近一次人工设置的"已下架"或"已上架"状态，不因显示为"已售出"而丢失。
- 若后续管理员在编辑页增加商品总数量（补货），导致剩余可售数量重新大于 0，则该商品的显示状态应立即恢复为 `base_status`。

### 4.2 修改状态弹层

多数量商品在**商品详情页**修改状态时唤起底部弹层：

| 元素 | 内容 |
|---|---|
| 标题 | 修改状态 |
| 当前商品 | 缩略图 + 名称 + "总数量 N / 剩余可售 M" |
| 目标状态 | 单选项：已下架 / 已上架 / 已售出（按业务规则过滤） |
| 本次修改数量 | 数值输入框 + ± 步进器（默认 1，最大值为剩余可售数量） |
| 主按钮 | "确认修改" |

### 4.3 🟩 业务规则

- **可操作数量** = 剩余可售数量 = 总数量 − 已售出 − 已结算。
- **本次修改数量** ≤ 可操作数量。
- 当前版本不拆分"已上架数量 / 已下架数量"；多数量商品仍只维护一个 `base_status`。
- 将商品改为"已下架"或"已上架"时，对整件商品的剩余可售部分生效，不单独选择数量。
- 只有改为"已售出"时需要填写"本次修改数量"。
- 各目标状态的语义：
  - 改为"已下架"：将 `base_status` 更新为已下架，剩余可售数量不变，但用户端不展示可售。
  - 改为"已上架"：将 `base_status` 更新为已上架，剩余可售数量不变，用户端展示可售。
  - 改为"已售出"：从剩余可售扣减 N 件，累加到"已售出数量"。当扣到剩余可售 = 0 时，整件商品的显示状态自动变为"已售出"。
- 修改为"已上架"需同时保证该商品至少存在 1 条合法购物平台链接（同 §1.5 / §6 编辑商品规则）。
- 已售出商品退款回退时，需填写"本次退款回退数量"，默认 1，最大值为当前 `soldCount`；确认后 `soldCount - N`，`base_status` 更新为已上架。

### 4.4 🟧 提示规则

| 场景 | 文案 | 形式 |
|---|---|---|

## 6. 编辑商品

### 6.1 🟦 显示规则

- 字段与单个上传一致（见 §2.1），所有字段预填当前数据。
- 顶部标题"编辑商品"，右上角"保存"按钮（灰态 / 常态遵循通用表单规则）。
- **可编辑字段**：包含**寄售用户**（允许在管理端更换寄售归属）；其他字段（IP / 系列 / 角色 / 类型 / 数量 / 价格 / 质量 / 购买记录 / 备注 / 购物平台链接 / 购买途径 / 照片）均可编辑。
- **不可编辑字段**：**商品状态**（在详情通过"修改状态"操作，不在编辑页改）。
- 页面顶部展示当前展示状态标签，但标签只读。
- 已结算商品进入编辑页时，直接 Toast "已结算商品不支持编辑"，并返回详情页。

🟩 寄售用户更换规则：
- 选择新寄售用户时，遵循"仅展示有寄售权限的用户"。
- 更换寄售用户后，**该商品的抽成比例**重新跟随**新用户的当前抽成**。
- 已售出 / 已结算商品**不允许更换寄售用户**（避免历史数据归属错乱）。
- 更换操作需 Dialog 二次确认："更换寄售用户后，该商品将归属新用户，且抽成按新用户的比例计算，是否继续？"

### 6.2 🟨 交互规则

- 进入编辑后跟踪是否有修改；如有未保存修改，返回时弹 Dialog 拦截。
- 字段校验规则、校验顺序、错误提示与"单个上传"一致。
- 编辑包含"购物平台链接组"字段；如商品当前为已上架且删空全部合法链接 → 阻断保存，就近提示。
- 编辑包含"购买途径"模块；如商品当前为已上架且没有任何启用展示的购买途径 → 阻断保存，就近提示。

### 6.2.1 购买途径编辑

购买途径模块放在“购物平台链接组”下方，标题为“用户端购买途径”。店铺名称与搜索关键词不在商品编辑页维护，统一从设置模块“店铺信息设置”读取。

| 元素 | 规则 |
|---|---|
| 平台行 | 固定展示淘宝/天猫、拼多多、小红书 3 行 |
| 展示开关 | 每个平台可单独开启 / 关闭用户端展示 |
| 店铺名称 | 只读展示全局店铺名称；来源于设置模块 |
| 搜索关键词 | 只读展示全局搜索关键词；来源于设置模块 |
| 内部购买链接 | 可选；填写时需符合对应平台链接规则；不在用户端展示 |
| 复制预览 | 展示用户端复制内容，便于管理员确认 |

🟩 业务规则：
- 管理员可在编辑商品页维护每个平台是否作为该商品的购买途径。
- 店铺名称与搜索关键词为全局配置，不允许在单个商品里单独修改；如需修改，进入设置模块“店铺信息设置”。
- 购买途径用于用户端“在哪里买”模块展示，不能直接把内部购买链接暴露给用户端。
- 商品已上架时，至少开启 1 个购买途径，且对应平台在全局店铺设置中为启用状态并已配置店铺名称。
- 若填写内部购买链接，平台必须与当前行一致；例如淘宝/天猫行只能填写淘宝或天猫域名。
- 如果仅开启平台、不配置内部购买链接，也允许保存并在用户端展示；用户可复制店铺名自行搜索，也可联系客服获取链接。
- 关闭某个平台展示后，该平台不出现在用户端“在哪里买”模块；内部购买链接可保留给客服使用。

### 6.3 🟩 业务规则

| 字段 | 校验 |
|---|---|
| IP / 系列 / 角色 | 字数上限 12，必填 |
| 自定义类型 | 字数上限 12，选"自定义"时必填 |
| 备注 | 字数上限 200 |
| 照片 | 最多 9 张，单张 ≤ 5M |
| 数量 | 大于 0；允许增加或减少（用于补货 / 纠错）；减少后不可低于"已售出 + 已结算"数量；增加后若剩余可售重新 > 0，则列表与详情状态恢复为 `base_status` |
| 价格 | 大于 0 |
| 购物平台链接 | 当商品已上架时至少 1 条必填；同平台不可重复；每条都需为白名单链接 |
| 购买途径 | 商品已上架时至少 1 个启用展示；启用展示的平台需在全局店铺设置中启用并配置店铺名称；内部购买链接不在用户端展示 |

🟩 保存规则：
- 保存时只更新发生变化的字段，并更新 `updatedAt`。
- 若修改总数量导致剩余可售数量从 0 变为大于 0，展示状态恢复为 `base_status`；若该 `base_status` 为已上架，则必须保留至少 1 条合法链接。
- 已售出商品允许编辑价格、质量、购买记录、备注、图片、链接、购买途径等展示信息；不允许修改寄售用户与减少总数量到低于历史数量。
- 商品编号、创建时间、已售出数量、已结算数量不可在编辑页修改。

### 6.4 🟧 提示规则

| 场景 | 文案 | 形式 |
|---|---|---|
| 保存成功 | 保存成功 | Toast，并退回详情 |
| 保存失败 | 数据保存失败，请稍后重试 | Toast |
| 校验失败 | 见 §2.4 校验清单 | 就近 |
| 修改为上架但未填链接 | 请至少填写 1 条购物平台链接后再上架 | 就近 |
| 已上架但未配置购买途径 | 请至少配置 1 个购买途径 | 就近 |
| 购买途径店铺名为空 | 请先到设置中维护店铺名称 | 就近 |
| 内部购买链接平台不匹配 | 链接平台与当前购买途径不一致 | 就近 |
| 返回拦截 | 当前修改尚未保存，确认离开吗？ | Dialog |
| 数量小于历史数量 | 数量不能小于已售出与已结算总数 | 就近 |
| 已结算进入编辑 | 已结算商品不支持编辑 | Toast |

## 7. 验收标准

### 7.1 商品列表

- 搜索 `角色 / IP / 系列 / 寄售用户 / 商品编号` 任一字段均可命中，搜索与筛选同时生效。
- 已售出 / 已结算商品在多选模式下不可勾选；全选只选中已下架 / 已上架商品。
- 批量上架只能选择已下架商品；每件商品补齐至少 1 条合法链接后才能提交。
- 批量下架只能选择已上架商品；确认后所有选中商品展示为已下架。
- 商品详情需展示最近 5 条操作记录，覆盖状态变更、编辑、结算操作。

### 7.2 上传与编辑

- 新建商品必填项缺失时，按 §2.4 顺序定位并提示。
- 单件上传页当前不展示购物平台链接字段。
- 单件上传页除备注外，当前展示字段均显示红色 `*`。
- 编辑商品页可维护淘宝/天猫、拼多多、小红书购买途径，包括展示开关和内部购买链接；店铺名称、搜索关键词从设置模块统一读取。
- 用户端展示购买途径时不得展示内部购买链接；只展示平台、店铺名称和搜索关键词。
- 已上架商品保存时至少保留 1 个启用展示的购买途径。
- 编辑总数量不能小于已售出数量与已结算数量之和。
- 已结算商品无法进入编辑、删除、修改状态、批量选择。

### 7.3 多数量状态

- 多数量商品部分售出后，只要剩余可售大于 0，展示状态仍跟随 `base_status`。
- 剩余可售扣为 0 时，展示状态自动变为已售出。
- 补货后剩余可售重新大于 0 时，展示状态恢复为 `base_status`。
- 已售出商品退款回退时可选择回退数量，回退数量不得超过当前已售出数量。

### 7.4 批量上传

- CSV / XLSX 模板缺少必需列时阻断，提示"模板列不匹配，请重新下载最新模板"。
- 表格中合法行可创建；IP / 角色 / 系列为空或价格 ≤ 0 的行跳过。
- 图片列按上传图片顺序编号匹配；图片编号无效时提示具体行号并停止导入。
- 批量上传当前不解析购物平台链接。
- 导入成功后 Toast 显示"导入成功 N 条"，并返回商品列表。
