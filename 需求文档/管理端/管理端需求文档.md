# 管理端总览与通用规则

## 文档说明

本文档用于记录"管理端"相关功能需求中的**总览信息与通用规则**。当前阶段已进入开发同步与需求迭代阶段，模块细节以拆分文档为准。

**文档范围**：仅覆盖**管理端**。所有功能默认由管理员执行；用户端（含寄售用户在 App 内的查看与操作）的需求另文记录，**本文不涉及**。

**完善目标**：每一条规则都应让开发、测试、设计**看完不需要再来追问**。每个功能点都需同时说清四个维度：

| 维度 | 解释 |
|---|---|
| 🟦 **显示规则** | 默认状态、置灰、高亮、空态、加载态、错误态长什么样，何时出现 / 消失 |
| 🟨 **交互规则** | 用户每一个动作（点击 / 输入 / 长按 / 滑动 / 清空）后系统给什么反馈 |
| 🟩 **业务规则** | 字段限制、字符长度、唯一性、不允许的情况、校验顺序 |
| 🟧 **提示文案** | 成功 / 失败 / 二次确认 / 空状态文案，以及提示形式（Toast / Dialog / 就近提示） |

**版本**：v2.1（功能优化补充版）
**关联原型**：`谷圈星社-寄售管理首页.html`、`谷圈星社-寄售管理简洁版.html`
**标注约定**：`[待确认]` 表示规则未定 / 需要业务方再确认；`[默认]` 表示采用通用规则。

**使用方式**：
- 后续设计、开发、测试默认**按模块文档推进**，不再直接基于这份总文档逐段执行。
- 本文档保留为**总览 + 通用基线 + 拆分导航**。

## 数据访问与权限基线

当前小程序通过 `dataAccess` 云函数统一封装部分集合的数据读写，前端通过 `utils/dataAccessService.js` 调用。

| 集合 / 数据 | 未登录用户 | 普通用户 | 寄售用户 | 管理员 |
|---|---|---|---|---|
| `products` 商品读取 | 仅可读取已上架商品 | 可读取商品数据 | 可读取商品数据 | 可读取商品数据 |
| `products` 商品写入 | ❌ | ❌ | ✅ 可管理自己寄售相关商品 | ✅ |
| `settlement_records` 结算记录 | ❌ | 仅本人记录 | 仅本人记录 | ✅ 全部 |
| `material_expenses` / `logistics_expenses` / `tech_service_expenses` | ❌ | ❌ | ❌ | ✅ |
| `admin_operation_logs` 操作日志 | ❌ | ❌ | ❌ | ✅ |
| `ip_groups` IP 分组 | ❌ | ❌ | ❌ | ✅ |

🟩 统一异常提示：
- 云函数未部署：提示"云函数未部署，请先上传 dataAccess 云函数"或对应 auth 云函数提示。
- 登录态失效：提示"登录态已失效，请重新登录"。
- 无权限：提示"暂无数据读取权限 / 暂无数据写入权限 / 暂无数据删除权限"。
- 网络异常：提示"网络异常，请稍后重试"。

# 通用规则

> 通用规则是所有功能模块的基线，下文每个功能均默认遵守，不再重复说明。如有差异，会在具体功能内单独标注。

## 1. 商品状态与状态流转

### 1.1 商品状态定义

| 状态 | 含义 |
|---|---|
| 已下架 | 商品已存在系统内，但未在用户端展示 |
| 已上架 | 商品在用户端展示，可被购买 |
| 已售出 | 商品已完成出售，等待结算 |
| 已结算 | 商品已完成结算，本流程结束 |

### 1.2 状态流转规则

**合法流转路径**：

```
新建 ─┬─→ 已下架 ⇄ 已上架 ─→ 已售出 ─→ 已结算（终态）
      └─→ 已上架            └─→ 已上架（仅退款回退）
```

| 当前状态 | 可流转到 | 不可流转到 |
|---|---|---|
| 已下架 | 已上架 | 已售出、已结算 |
| 已上架 | 已下架、已售出 | 已结算（必须先经已售出） |
| 已售出 | 已结算、已上架（仅退款回退） | 已下架 |
| 已结算 | —（终态） | 任何状态 |

🟧 **提示**：
- 尝试非法流转 → Toast 提示 "当前商品状态不支持该操作"。
- 已结算商品在所有列表、详情中均**置灰显示**，不响应选择 / 编辑 / 删除 / 修改状态。
- 退款回退属于**异常售后流转**，仅允许在商品详情页对已售出商品单独执行，回退后商品状态恢复为**已上架**。

### 1.3 商品状态对应操作权限矩阵

| 操作 | 已下架 | 已上架 | 已售出 | 已结算 |
|---|---|---|---|---|
| 选择（多选 / 全选） | ✅ | ✅ | ❌ | ❌ |
| 编辑商品 | ✅ | ✅ | ✅ | ❌ |
| 删除商品 | ✅ | ✅ | ❌ | ❌ |
| 修改商品状态 | ✅ | ✅ | ✅（详情页支持退款回退为已上架；"已售出 → 已结算"由结算页触发） | ❌ |
| 进入结算 | ❌ | ❌ | ✅ | ❌ |

> "全选"时自动跳过 ❌ 的状态。

## 2. 通用列表规则

### 2.1 🟦 显示规则

- **默认排序**：所有列表统一按"添加 / 注册时间倒序"。商品、记录、日志等按添加时间倒序；用户列表按注册时间倒序。
- **分页方式**：上拉加载更多。每页 20 条。`[默认]`
- **加载态**：
  - 首次进入：列表区域显示骨架屏 3~5 条。
  - 加载更多：列表底部显示"正在加载…"。
  - 全部加载完：列表底部显示"没有更多了"。
- **空状态**：
  - 列表无数据 → 居中显示插画 + 文案，文案见各模块。
  - 搜索 / 筛选无结果 → 显示文案"未找到匹配结果"，并提供"清空筛选"按钮。
- **错误态**：
  - 列表加载失败 → 居中显示"加载失败，点击重试"按钮，点击后重新请求。

### 2.2 🟨 交互规则

- **下拉刷新**：从列表顶部下拉超过阈值触发，刷新当前页数据；刷新中显示加载图标，完成后 Toast 提示"已刷新"。
- **点击列表项**：进入详情页。
- **长按列表项**：不触发多选模式，也不唤起任何批量操作入口；多选仅通过列表顶部"多选"按钮进入。
- **多选模式**：
  - 顶部右侧出现"多选"按钮（图标 / 文字），点击后进入多选模式。
  - 多选模式下：列表项左侧出现复选框；顶部出现"全选 / 取消全选"切换；底部出现批量操作栏。
  - 退出多选：点击右上角"取消"或物理返回键。

### 2.3 🟩 业务规则

- 已结算商品：列表中**置灰**、复选框**禁用**。
- 已售出商品：在"全部列表"中可点击查看，但复选框**禁用**；在"已出售列表"中复选框**可用**（用于结算选择）。
- 全选支持**跨页**：点击"全选"后，勾选当前筛选结果下的全部可操作商品；翻页后已勾选项持续保留，直到点击"取消全选"或退出多选模式。

## 3. 通用搜索与筛选规则

### 3.1 搜索框

| 项 | 规则 |
|---|---|
| 🟦 默认提示 | 焦点前 / 后均显示提示文案（各模块见自身定义） |
| 🟦 清空按钮 | 输入框存在内容时，右侧出现"✕"清空按钮；点击后一键清空 |
| 🟨 输入触发 | 输入停止 500ms 后自动搜索；亦支持回车 / 键盘搜索键提交 |
| 🟨 搜索历史 | 不保留搜索历史 |
| 🟧 空结果 | 显示"未找到匹配结果"，并展示"清空搜索"按钮 |

### 3.2 筛选器

| 项 | 规则 |
|---|---|
| 🟦 默认状态 | 未选中任何条件时，筛选按钮显示为常态；选中条件时按钮显示"●"红点 + 已选数量 |
| 🟨 应用 | 点击"应用"后关闭筛选面板并刷新列表 |
| 🟨 重置 | 点击"重置"清空当前面板内所有选项；不自动关闭面板 |
| 🟧 应用成功 | 列表顶部出现已选条件 Chips，点击 Chips 上的"✕"可单独移除该条件 |

## 4. 通用表单规则

### 4.1 字段层

| 项 | 规则 |
|---|---|
| 🟦 必填标识 | 字段标签前显示红色 "*" |
| 🟦 默认提示 | 焦点前 / 焦点后均显示占位提示（各字段自有文案） |
| 🟦 字符计数 | 设置了字符上限的字段，右下角显示"已输入 / 上限"，超出上限时显示红色 |
| 🟦 清空按钮 | 输入框存在内容时右侧出现"✕"，点击清空 |
| 🟦 错误态 | 校验失败时，输入框红色描边 + 字段下方就近显示红色错误文案 |
| 🟨 校验时机 | 失焦时进行单字段校验；提交时进行整体校验 |
| 🟩 字符限制 | 超出字符上限时**不可继续输入**（直接截断 / 阻断输入） |

### 4.2 主操作按钮

| 项 | 规则 |
|---|---|
| 🟦 灰态 | 任一必填项为空时，主按钮置灰、不响应点击 |
| 🟦 常态 | 全部必填项有值时，主按钮恢复主色 |
| 🟦 加载态 | 点击后按钮显示菊花图标 + 文案"提交中…"，按钮不可再次点击 |
| 🟨 校验顺序 | 自上而下逐字段校验，遇到首个失败立即停止，定位到该字段并显示错误 |

### 4.3 离开拦截

- 表单存在未保存修改时，点击"返回 / 关闭 / 系统返回手势"→ Dialog 二次确认：**"当前修改尚未保存，确认离开吗？"** [取消 / 确认离开]

## 5. 二次确认弹窗规则

适用于：删除商品、批量删除、修改商品状态、批量修改状态、重置密码、修改抽成、提交结算、导出数据。

| 元素 | 规则 |
|---|---|
| 标题 | 简短描述操作（如"删除商品"） |
| 内容 | **说明影响范围**：如"删除后不可恢复，确认删除该商品吗？"或"本次将影响 12 件商品" |
| 按钮 | 左"取消"（次按钮，灰底）+ 右"确认"（主按钮，主色） |
| 危险操作 | 删除类的"确认"按钮使用**红色**（如删除、重置密码） |
| 关闭方式 | 点击遮罩 / 取消按钮 / 物理返回键 均关闭弹窗，操作中断 |

## 6. 反馈提示样式规则

| 类型 | 样式 | 何时使用 | 持续时间 |
|---|---|---|---|
| Toast | 屏幕中部短提示 | 操作成功 / 简短失败 / 一般信息 | 1.5s 自动消失 |
| 就近提示 | 字段下方红色文案 | 字段级校验失败 | 直至该字段重新校验通过 |
| Dialog | 居中弹窗 + 按钮 | 二次确认 / 阻塞性错误 | 用户操作后关闭 |
| 顶部 Banner | 列表上方横条 | 网络异常 / 系统公告 | 用户关闭或恢复后消失 |
| Snackbar | 屏幕底部条状提示 + 可点击操作 | 当前版本不启用；如后续增加撤销类操作再启用 | 3s 自动消失 |

### 6.1 标准提示文案库

**成功类（Toast）**

| 场景 | 文案 |
|---|---|
| 上传 | 上传成功 |
| 保存 | 保存成功 |
| 删除 | 删除成功 |
| 状态修改 | 状态修改成功 |
| 结算 | 结算成功 |
| 重置密码 | 重置密码成功 |
| 导出 | 导出成功 |
| 备份 | 备份成功 |

**失败类（Toast 或就近提示）**

| 场景 | 文案 | 形式 |
|---|---|---|
| 必填未填 | 请填写 [字段名] | 就近 |
| 字数超限 | [字段名] 字数不能超过 [N] 个 | 就近 |
| 图片超大 | 图片大小不能超过 5M | Toast |
| 图片超量 | 图片最多上传 [N] 张 | Toast |
| 未选商品 | 请先选择商品 | Toast |
| 状态不支持 | 当前商品状态不支持该操作 | Toast |
| 无寄售权限 | 当前用户无寄售权限 | Toast |
| 保存失败 | 数据保存失败，请稍后重试 | Toast |
| 加载失败 | 数据加载失败，请刷新后重试 | Banner / 错误态 |

**空状态类**

| 场景 | 文案 |
|---|---|
| 商品列表为空 | 暂无商品 |
| 用户列表为空 | 暂无用户 |
| 统计为空 | 暂无统计数据 |
| 结算记录为空 | 暂无结算记录 |
| 搜索 / 筛选无结果 | 未找到匹配结果 |
| 操作日志为空 | 暂无操作记录 |

## 7. 通用异常处理

| 异常 | 提示 | 形式 |
|---|---|---|
| 网络断开 | 网络异常，请检查后重试 | 顶部 Banner（持续显示，恢复后自动隐藏） |

## 8. 通用显示格式

| 项 | 格式 | 示例 |
|---|---|---|
| 金额 | 人民币 ¥，千分位逗号，保留两位小数（整数时省略） | ¥ 7,770 / ¥ 1,280.50 |
| 百分比 | 整数 + %，不展示小数位 | 10% / 25% |
| 时间（相对） | 1 小时内：N 分钟前；24 小时内：N 小时前；昨天：昨天 HH:mm；7 天内：N 天前；超过：YYYY-MM-DD | 2 小时前 / 昨天 14:32 |

| 日期段 | YYYY-MM-DD 至 YYYY-MM-DD | 2026-05-01 至 2026-05-26 |
| 商品编号 | "#" + 6 位字母数字 | #A00128 |
| 用户 ID | 平台分配的全局唯一字符串 | U10001 |
