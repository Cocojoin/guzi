# 三、用户管理

用户管理用于查看和管理平台内寄售用户的信息、抽成比例、寄售权限和商品记录。

## 0. 当前开发同步说明

截至 2026-06-05，用户管理主功能集中在 `admin/pages/users/users`，通过页面内多视图切换实现：

| 视图 | 已实现能力 |
|---|---|
| 用户列表 | 搜索、角色筛选、用户卡片、用户统计数据 |
| 用户详情 | 昵称、联系方式、抽成编辑；寄售权限开关；重置密码；商品 / 待结算 / 已结算入口 |
| 用户商品 | 展示当前用户全部商品及状态 |
| 待结算 | 展示当前用户未结算售出批次，默认选中，支持进入结算 |
| 结算 | 汇总商品总额、平台抽成、应付金额，填写实际收入后提交 |
| 已结算 | 展示历史结算记录与结算详情 |

`admin/pages/users/users-detail` 当前未配置在 `app.json` 中，视为历史静态详情页，不作为本期主流程入口。

## 2. 用户详情

### 2.1 🟦 显示内容

**Header 区**

| 元素 | 规则 |
|---|---|
| 头像 | 大尺寸（96×96） |
| 昵称 | 主标题 |
| 账号 | 副标题，灰色 |

**信息区（可编辑字段含 "✎" 图标）**

| 字段 | 是否可编辑 | 显示格式 |
|---|---|---|
| 昵称 | ✅ | 文本 |
| 联系方式 | ✅ | 文本（电话 / 微信号 / 其他，无格式校验） |
| 抽成 | ✅ | 百分比（0%~100%） |
| 账号 | ❌ | 只读 |

**统计区（卡片）**

| 卡片 | 内容 | 点击效果 |
|---|---|---|
| 寄售商品数量 | 该用户全部商品总数 | 进入该用户的"寄售商品列表" |
| 已出售商品数量 | 已售出但未结算的商品数量 | 进入"已出售商品列表"（可结算） |
| 已结算商品数量 | 历史已结算商品总数 | 进入"已结算商品记录列表" |

**操作区**

| 操作 | 说明 |
|---|---|
| 重置密码按钮 | 见 §2.3 |
| 寄售权限开关 | 见 §2.4 |

### 2.2 🟨 交互规则

- 点击 "✎" 或字段本身进入编辑态：字段变为输入框，右上角"保存 / 取消"按钮出现。
- 修改"抽成"前，弹 Dialog 二次确认（见 §2.5 抽成规则）。
- 点击数量卡片 → 在当前页面内切换到对应商品 / 待结算 / 已结算视图。

### 2.3 重置密码

🟦 显示规则：
- 操作区显示"重置密码"按钮，主色描边 + 红色文字（次危险操作）。

🟨 交互规则：
1. 点击按钮 → Dialog 二次确认："确认将该用户密码重置为 123456 吗？"
2. 确认 → 按钮加载态，请求服务端。
3. 成功 → Toast "重置密码成功"。
4. 失败 → Toast "操作失败，请稍后重试"。

🟩 业务规则：
- 重置后密码固定为 `123456`。
- 当前版本不强制用户下次登录修改密码。

### 2.4 寄售权限开关

🟦 显示规则：
- 开关默认"关 / 开"反映服务端当前状态。
- 开关为"关"时，用户详情头部增加"已停用寄售"标签。

🟨 交互规则：

| 当前状态 | 操作 | Dialog 文案 |
|---|---|---|
| 开 → 关 | 点击关闭 | "关闭后，该用户已上架商品将全部自动下架，确认关闭吗？" |
| 关 → 开 | 点击开启 | "确认开启该用户的寄售权限吗？" |

🟩 业务规则：

| 关闭前的商品状态 | 关闭后的处理 |
|---|---|
| 已上架 | 当前开发仅关闭用户寄售权限，不自动改动商品状态；自动下架作为后续增强 |
| 已下架 | 不变 |
| 已售出 | 不变；后续仍可正常进入结算流程 |
| 已结算 | 不变 |

- 关闭后，用户**不能在用户端提交新的商品寄售**。
- 重新开启权限后，用户恢复在用户端提交寄售的能力。
- 若后续实现自动下架，本质上等同于"批量已上架 → 已下架"，需遵循通用状态流转规则并写入操作日志。

🟧 提示：

| 场景 | 文案 | 形式 |
|---|---|---|
| 关闭权限前 | 关闭后，该用户已上架商品将全部自动下架，确认关闭吗？ | Dialog |
| 关闭权限成功 | 已关闭寄售权限 | Toast |
| 关闭权限但无在售商品 | 已关闭寄售权限 | Toast |
| 开启权限前 | 确认开启该用户的寄售权限吗？ | Dialog |
| 开启权限成功 | 已开启寄售权限 | Toast |

### 2.5 抽成规则

🟦 显示规则：
- 抽成字段输入框右侧附 "%" 后缀。
- 输入仅允许 0~100 **整数**，不支持小数。

🟨 交互规则：
1. 编辑抽成 → 点击保存。
2. 校验通过后 Dialog 二次确认："修改抽成后，仅影响未售出和未结算商品，确认修改吗？"
3. 确认 → 请求服务端。
4. 成功 → Toast "保存成功"。

🟩 业务规则：

| 商品状态 | 抽成变更后采用的比例 |
|---|---|
| 已下架 | **新**比例 |
| 已上架 | **新**比例 |
| 已售出 | **原**比例（即售出当下的比例） |
| 已结算 | **原**比例（已结算不可变） |

🟧 提示：

| 场景 | 文案 | 形式 |
|---|---|---|
| 抽成为空 | 请填写抽成比例 | 就近 |
| 抽成格式错（非数字 / 超 100%） | 请填写正确的抽成比例 | 就近 |
| 修改前 | 修改抽成后，仅影响未售出和未结算商品，确认修改吗？ | Dialog |
| 修改成功 | 保存成功 | Toast |

### 2.6 字段校验汇总

| 字段 | 校验 | 失败提示 |
|---|---|---|
| 昵称 | 必填，字数上限 20 | 请填写用户昵称 / 昵称字数不能超过 20 个 |
| 联系方式 | 必填，字数上限 50 | 请填写联系方式 |
| 抽成 | 必填，0~100 整数 | 请填写抽成比例 / 请填写正确的抽成比例 |

🟨 校验顺序：昵称 > 联系方式 > 抽成。

## 4. 结算页面

结算页面用于对已选择的"已售出"商品进行结算。

### 4.1 🟦 显示内容

**顶部信息**
- 寄售用户：头像 + 昵称
- 本次结算商品数量：N 件

**商品列表（每件）**

| 字段 | 显示 |
|---|---|
| 图片 | 缩略图 |
| 商品名称 / 角色 | 主标题 |
| IP / 系列 | 副标题 |
| 商品价格 | ¥ + 千分位 |
| 商品数量 | 本次结算数量（多数量商品时为已售出部分的数量） |

**汇总区（实时计算）**

| 字段 | 计算 | 说明 |
|---|---|---|
| 商品总额（寄售价） | sum(商品价格 × 数量) | 用户标价合计 |
| 抽成比例 | 取自各商品**售出时**的抽成；若所选商品的历史抽成不同，分组显示并标注（见 §4.3） | — |
| 平台抽成金额 | sum(商品价格 × 数量 × 抽成比例) | 平台从抽成获得的钱 |
| **应付寄售用户金额** | 商品总额 − 平台抽成金额 | 主色高亮，**即本次需支付给该寄售用户的钱** |
| 商品实际收入 | 管理员手动填写（必填） | 在外部平台实际成交金额；仅平台统计用，**不影响应付寄售用户金额** |

**凭证上传区**
- 图片网格，最多 3 张；每张 ≤ 5M。
- 用于记录转账凭证。

**主按钮**：提交结算

### 4.2 🟨 交互规则

| 操作 | 效果 |
|---|---|
| 修改商品实际收入输入框 | 失焦时校验：必须 > 0、数值合法 |
| 上传凭证 | 点击 "+" 调起相册 / 拍照；超过 3 张时 "+" 隐藏 |
| 移除凭证 | 缩略图右上角"✕" |
| 提交 | Dialog 二次确认："确认提交本次结算吗？提交后所选商品状态变为已结算且不可撤销" |
| 提交成功 | Toast "结算成功"；返回上一页（已出售商品列表），列表刷新 |

### 4.3 🟩 计算与抽成规则

#### 金额计算

- **商品总额（寄售价）** = sum(商品价格 × 数量)，是寄售用户标价的合计。
- **平台抽成金额** = sum(商品价格 × 数量 × 抽成比例)。
- **应付寄售用户金额** = 商品总额 − 平台抽成金额 = sum(商品价格 × 数量 × (1 − 抽成比例))。
  - 这是本次结算**实际需要转账给寄售用户的钱**。
  - **完全按"商品价格"计算，与"商品实际收入"无关**。
- **商品实际收入**：管理员手动填写在外部购物平台的真实成交价合计。
  - 仅作为平台内部统计依据（用于核算"平台差价收益"）。
  - **不展示在用户端**，**不参与给寄售用户的结算金额计算**。

#### 抽成比例的取值

每件商品的抽成比例**锁定在售出时**：

| 用户抽成变更场景 | 本次结算使用的抽成比例 |
|---|---|
| 用户在售出前从未修改过抽成 | 直接使用售出时的当前比例（即用户当前比例） |
| 用户在售出后修改过抽成 | 仍使用**售出时**的比例（不跟随新比例） |
| 同一批次结算中存在多个商品，售出时抽成不同 | 分组计算各自抽成金额，再汇总 |

> 设计意图：抽成比例锁定在售出时刻，避免管理员通过事后改抽成影响已售出商品的结算金额，保护寄售用户利益。

#### 字段可见性

| 字段 | 管理端结算页面 | 用户端 |
|---|---|---|
| 商品价格 | ✅ | ✅ |
| 抽成比例 | ✅ | ❌ 不展示 |
| 平台抽成金额 | ✅ | ❌ 不展示 |
| 应付寄售用户金额 | ✅ | ✅（用户能看到自己拿到多少） |
| 商品实际收入 | ✅ | ❌ 不展示 |

### 4.4 🟧 提示规则

| 场景 | 文案 | 形式 |
|---|---|---|
| 未选商品 | 请至少选择一件商品进行结算 | Toast |
| 实际收入为空 | 请填写商品实际收入 | 就近 |
| 实际收入 ≤ 0 | 商品实际收入必须大于 0 | 就近 |
| 未上传凭证 | 请上传结算凭证 | 就近 |
| 凭证超 3 张 | 最多上传 3 张凭证 | Toast |
| 单张凭证超 5M | 单张凭证不能超过 5M | Toast |
| 提交前 | 确认提交本次结算吗？提交后所选商品状态变为已结算且不可撤销 | Dialog |
| 结算成功 | 结算成功 | Toast |
| 结算失败 | 结算失败，请稍后重试 | Toast |

### 4.5 结算后果

- 所选商品状态变为**已结算**。
- 自动进入该用户的"已结算商品记录列表"。
- 已结算商品**不再支持**删除、修改状态、选择、编辑。

## 5. 已结算商品记录列表

### 5.1 🟦 显示规则

- 列表按结算时间倒序展示；当前版本不做按月折叠。
- 每条记录显示：

| 字段 | 显示 |
|---|---|
| 结算日期 | YYYY-MM-DD |
| 商品数量 | "共 N 件" |
| 商品总额 | ¥ + 千分位 |
| 抽成金额 | ¥ + 千分位（灰色辅助） |

### 5.2 🟨 交互规则

- 点击记录 → 进入"结算记录详情"。
- 下拉刷新 / 上拉加载更多遵循通用列表规则。

### 5.3 结算记录详情

🟦 显示内容：

| 区块 | 内容 |
|---|---|
| 顶部 | 结算日期 + 商品总数 |
| 商品列表 | 本次结算的所有商品（缩略图 + 名称 + 价格 + 数量） |
| 金额汇总 | 商品总额 / 抽成金额 / 商品实际收入 |
| 凭证 | 凭证图片缩略图，可点击全屏预览 |

🟨 交互规则：
- 点击商品 → 进入商品详情（已结算形态，无操作按钮）。
- 点击凭证 → 全屏预览，支持左右滑动。

🟧 空态：暂无结算记录。
