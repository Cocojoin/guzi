# 一、统计

统计模块用于查看平台经营数据，重点关注**已结算商品收入**、**平台抽成收入**、**材料费用支出**、**物流费用支出**。

## 0. 当前开发同步说明

截至 2026-06-05，统计模块已在 `admin/pages/stats/stats` 内以多视图方式实现：

| 视图 | 已实现能力 |
|---|---|
| 统计首页 | 本月 / 累计切换、月份面板、商品状态数量、Hero 查看明细入口、抽成、差价、总支出、查询明细卡、生成寄售月报入口 |
| 寄售月报 | 按月份生成宣传海报预览，支持保存图片 |
| 收入明细 | 基于 `settlement_records` 展示已结算收入，支持本月 / 近三月 / 自定义 / 全部筛选与关键词搜索 |
| 材料费用 | 材料费用列表、月度 / 累计切换、新增 / 编辑 / 删除、凭证图片 |
| 物流费用 | 物流费用列表、新增 / 编辑 / 删除 |
| 统计汇总 | 基于累计数据展示收入、支出、净收入等公式拆解 |

## 1. 金额概念与收入口径

### 1.1 金额概念（重要）

> 统计与结算涉及两个**不同**的金额，开发与测试需先理解二者区别，所有公式都建立在该定义之上。

| 金额 | 含义 | 由谁定义 | 用途 |
|---|---|---|---|
| **商品价格**（寄售价格） | 寄售用户在管理端上传商品时填写的标价 | 寄售用户 / 管理员代填 | 1. 用户端展示价；2. **结算给寄售用户的依据**；3. 计算平台抽成的依据 |
| **商品实际收入** | 管理员将商品挂到外部购物平台（淘宝 / 拼多多 / 小红书）后**实际成交的价格** | 管理员在结算时手动填写 | 仅用于平台内部统计，**不影响**给寄售用户的结算金额；用于让管理员了解平台真实盈亏 |

**两者关系**：
- 实际收入 > 商品价格 → 平台多赚差价
- 实际收入 < 商品价格 → 平台从抽成中贴补差价
- 实际收入 = 商品价格 → 平台仅获得抽成收益

🟦 显示原则：
- "商品实际收入"字段**仅在管理端可见**（结算页面录入、统计页面展示），**不在用户端任何位置展示**。
- 寄售用户在自己的结算记录中看到的金额，是按**商品价格**计算的结算金额。

### 1.2 收入口径

| 项 | 规则 |
|---|---|
| 统计对象 | **仅已结算商品** |
| 统计金额 | 已结算商品的**商品价格**与**商品实际收入**同时计入统计 |
| 排除范围 | 已下架、已上架、已售出商品均**不**计入收入统计 |
| 统计维度 | 日 / 周 / 月 / 自定义时间范围 |
| 平台抽成 | 每个用户的抽成比例可不同，抽成基数为**商品价格**（非实际收入） |
| 抽成变更影响 | 用户中途修改抽成比例后：**已售出和已结算商品**仍按**售出时**的抽成比例；其他状态商品按**新**抽成比例 |
| 给寄售用户的结算金额 | 商品价格 × (1 − 抽成比例)，**与商品实际收入无关** |
| 货币单位 | 人民币 ¥ |
| 退款影响 | 若商品在**已售出未结算**阶段发生退款，则商品状态从"已售出"回退为"已上架"，退出待结算口径，也**不计入平台抽成收入**；统计侧不做单独"扣减"动作 |

## 2. 收入展示（统计首页 Hero 卡 + 4 宫格）

### 2.1 🟦 显示规则

**Hero 主卡（顶部）**

| 元素 | 规则 |
|---|---|
| 主标题 | 默认为"本月实际收益"；切换 Tab 后变为"累计实际收益" |
| 主金额 | ¥ + 千分位金额，显著字号 |
| 月份选择按钮 | 标题右侧；显示当前月份 YYYY-MM 与下拉箭头 |
| 查看明细按钮 | 主标题右侧展示"查看明细 >"胶囊按钮，点击进入收入明细 |
| 趋势 Chip | "较上月 +N%"（涨）或"较上月 -N%"（跌），上升用 ↑ 图标，下降用 ↓ 图标 |
| 累计模式下 | 月份按钮**置灰、不可点击**；趋势 Chip 隐藏 |

**4 宫格统计磁贴**

| 磁贴 | 含义 | 取数 |
|---|---|---|
| 总商品 | 全部商品总数（含所有状态） | count(all) |
| 在售 | 当前已上架商品数 | count(已上架) |
| 已出售 | 当前已售出商品数 | count(已售出) |
| 已结算 | 累计已结算商品数 | count(已结算) |

🟦 数字进入页面时从 `0` 滚动到目标值（动画时长 ~1.1s）。

**查询明细卡（本月 / 累计 切换）**

| 卡片 | 显示 | 含义 | 计算 |
|---|---|---|---|
| 实际销售额 | 单行大卡，右侧箭头 | 已结算商品的**商品价格**之和 | sum(price) |
| 结算金额 | 半宽卡，右侧箭头 | 已结算商品的**商品实际收入**之和 | sum(actual_income) |
| 费用支出 | 半宽卡，右侧箭头，金额红色显示，前缀 "-" | 材料费用 + 物流费用之和 | sum(material) + sum(logistics) |

🟩 调整说明：
- 统计首页底部不再展示"实际收入（平台净收入）"蓝色汇总条。
- 平台净收入仍作为 Hero 主金额展示，并保留在统计汇总关系中用于财务核对；但不再作为统计首页底部蓝色汇总条重复展示。
- 查询明细卡用于承接图中"查看明细"入口，点击后进入对应明细页或筛选结果。

**生成寄售月报入口卡**

| 元素 | 规则 |
|---|---|
| 位置 | 统计首页 Hero 与商品数量磁贴下方，查询明细卡上方 |
| 标题 | "生成寄售月报" |
| 副标题 | "一键生成宣传海报 · 保存分享拉新" |
| 左侧图标 | 使用上传 / 图片生成类图标，粉紫渐变底 |
| 右侧箭头 | 点击进入月报生成页 |
| 展示条件 | 仅在"本月"模式下展示；累计模式下隐藏或置灰不可点 |

### 2.2 🟨 交互规则

| 操作 | 效果 |
|---|---|
| 点击 Hero 月份按钮 | 弹起底部"月份选择"面板（年份切换 + 12 月网格） |
| 月份面板：选中某月 | 关闭面板，Hero 与统计卡数据切换到该月，Toast 提示"已切换至 YYYY 年 N 月" |
| 月份面板：未来月份 | 透明度 35%，不可点击 |
| 月份面板：年份切换 | 上月按钮 / 下月按钮；当年份达到当前年时下一年按钮置灰 |
| 切换 Tab "累计统计" | 月份按钮置灰；统计卡切换为累计口径数据 |
| 点击磁贴 | 跳转到对应分类的商品列表（如点击"在售"→ 商品列表筛选为已上架） |
| 点击 Hero "查看明细" | 进入收入明细页，并带入当前月份或累计筛选条件 |
| 点击"实际销售额"卡 | 进入收入明细页，并带入当前月份或累计筛选条件 |
| 点击"结算金额"卡 | 进入收入明细页，并带入当前月份或累计筛选条件 |
| 点击"费用支出"卡 | 进入费用支出明细视图，展示材料费用与物流费用记录 |
| 点击"生成寄售月报" | 进入寄售月报页，默认带入当前统计首页选中的月份 |

### 2.3 🟩 业务规则

- 时间范围为空时，默认展示**全部统计数据**（即累计模式）。
- 切换月份时，所有数据按**该月内数据全量重算**，不采用月末快照口径。
- 实际收益使用 §7 中"平台净收入"的计算公式。
- 统计首页不再以底部蓝色汇总条重复展示平台净收入；如需核对公式，进入统计汇总关系或明细核对。

### 2.4 🟧 提示

| 场景 | 文案 |
|---|---|
| 数据加载中 | 各数据行显示骨架占位 |
| 数据为空 | 显示"暂无统计数据" |
| 切换成功 | "已切换至 YYYY 年 N 月" |
| 月报数据不足 | "当前月份暂无可生成月报的数据" |

## 3. 生成寄售月报

寄售月报用于把管理端统计数据生成一张可保存的宣传图片，便于管理员在社群、朋友圈或平台账号中分享拉新。

### 3.1 页面入口

| 入口 | 规则 |
|---|---|
| 统计首页入口卡 | 点击"生成寄售月报"进入 |
| 默认月份 | 继承统计首页当前选择月份 |
| 路由建议 | `admin/pages/stats/month-report/month-report`；如不新增路由，也可作为 `admin/pages/stats/stats` 内部子视图实现 |

### 3.2 🟦 页面结构

| 区块 | 内容 |
|---|---|
| 顶部导航 | 返回按钮 + 标题"寄售月报" + 副标题"生成图片 · 用于宣传分享" |
| 月份选择 | 右上角月份下拉，如"5 月"；支持切换历史月份 |
| 海报预览 | 中间展示完整月报卡片，所见即保存图片内容 |
| 底部按钮 | 固定底部主按钮"保存图片" |

### 3.3 🟦 月报海报内容

海报为一张竖版长图，整体视觉沿用蓝粉渐变、玻璃卡片和圆角风格。

| 区块 | 展示内容 | 取数 / 规则 |
|---|---|---|
| 店铺信息 | 店铺 Logo / 店铺名 / 账号名 / 业务说明 | 默认展示"谷子寄售小铺"、`@guzi_shop`、"谷圈寄售代售"；后续可接设置项 |
| 月报标签 | "寄售月报" + `YYYY 年 M 月` | 根据当前选择月份生成 |
| 核心成交卡 | "本月帮谷友成交" + 金额 | 本月已结算商品的商品价格合计，即实际销售额 |
| 趋势 Chip | "较上月 +N%" / "较上月 -N%" | 本月实际销售额与上月实际销售额对比；上月为 0 时展示"本月新增" |
| 成交说明 | "成交 N 件 · 已全部为谷友完成结算" | N 为本月已结算商品数量 |
| 寄售 IP | 寄售 IP 数 | 本月已结算商品中 IP 去重数量 |
| 平台总商品 | 平台总商品数 | count(all products) |
| 当前在售 | 当前已上架商品数 | count(已上架) |
| 本月成交 | 本月已结算商品数量 | count(settled in selected month) |
| 寄售谷友 | 寄售用户总数 + 本月新增 | 有寄售权限用户数；本月新增按开通寄售权限时间或用户创建时间统计 `[按现有数据可取字段实现]` |
| 小程序用户 | 小程序用户总数 + 本月新增 | 全部用户数；本月新增按用户创建时间统计 |
| 宣传 CTA | "把你的谷子交给我们寄售" | 固定文案 |
| CTA 副文案 | "省心代售 · 透明结算" | 固定文案 |
| 小程序码 | 小程序码或占位二维码 | 用于扫码进入小程序开通 |
| 页脚 | `数据由「谷子寄售小铺」于 YYYY-MM-DD 生成 · 仅供参考` | 日期取生成当天 |

### 3.4 🟨 交互规则

| 操作 | 效果 |
|---|---|
| 点击返回 | 返回统计首页，保留原月份 |
| 点击月份下拉 | 弹出月份选择面板，规则同统计首页月份选择 |
| 切换月份 | 重新计算并刷新海报预览 |
| 点击保存图片 | 将海报预览生成图片并保存到系统相册 |
| 保存前未授权相册 | 调起授权；拒绝后提示用户开启相册权限 |

🟩 重点规则：
- 底部主按钮文案固定为"保存图片"，不使用"转发"作为主按钮文案。
- 不在页面底部展示"长按图片即可保存 / 转发"作为主操作；保存动作由按钮触发。
- 保存图片内容只包含海报预览区域，不包含管理端底部 TabBar、顶部系统状态栏和页面底部按钮。
- 月报为宣传图，不作为财务结算凭证；页脚需展示"仅供参考"。

### 3.5 🟩 取数规则

| 数据 | 口径 |
|---|---|
| 本月帮谷友成交金额 | 选中月份内已结算商品的商品价格合计 |
| 本月成交件数 | 选中月份内已结算商品数量 |
| 寄售 IP | 选中月份内已结算商品的 IP 去重数量 |
| 平台总商品 | 当前商品总数，包含全部状态 |
| 当前在售 | 当前已上架商品数 |
| 寄售谷友 | 当前有寄售权限的用户总数 |
| 寄售谷友本月新增 | 选中月份内新增寄售权限用户数；若无开通时间字段，则使用用户创建时间兜底 |
| 小程序用户 | 当前用户总数 |
| 小程序用户本月新增 | 选中月份内新增用户数 |
| 较上月百分比 | `(本月成交金额 - 上月成交金额) / 上月成交金额`，四舍五入取整数百分比 |

### 3.6 🟧 提示规则

| 场景 | 文案 | 形式 |
|---|---|---|
| 数据加载中 | 生成中 | Loading |
| 当前月份无成交 | 当前月份暂无可生成月报的数据 | Toast 或空态 |
| 保存中 | 正在保存图片 | Loading |
| 保存成功 | 图片已保存到相册 | Toast |
| 保存失败 | 保存失败，请稍后重试 | Toast |
| 相册权限被拒绝 | 请开启相册权限后再保存 | Toast |
| 小程序码加载失败 | 小程序码加载失败，请稍后重试 | 就近提示或占位 |

## 4. 收入明细

收入明细用于查看已结算商品收入来源，方便核对每一笔收入。

### 4.1 🟦 显示字段

| 字段 | 说明 | 格式 |
|---|---|---|
| 商品编号 | 系统编号 | #A00128 |
| 商品名称 | 角色 / IP / 系列拼接或主标题 | — |
| 寄售用户 | 用户昵称（点击可跳转用户详情） | — |
| 商品价格 | 商品标价 | ¥ + 千分位 |
| 商品实际收入 | 实际成交收入 | ¥ + 千分位 |
| 平台抽成比例 | 结算时的抽成比例 | 10% |
| 平台抽成金额 | 商品价格 × 抽成比例 | ¥ + 千分位 |
| 商品状态 | 固定显示"已结算"（明细已经过滤） | 标签 |
| 备注 | 结算备注 | 文本，长则截断 + "…" |
| 结算时间 | 结算操作时间 | YYYY-MM-DD HH:mm |

### 4.2 筛选条件

- 时间范围（日历选择器，支持快捷"今日 / 本周 / 本月 / 上月 / 自定义"）
- 商品编号（精确匹配）
- 商品名称（模糊匹配）
- 用户昵称或用户 ID（模糊匹配）

### 4.3 🟨 交互规则

- 点击明细行 → 进入**商品详情**页（详情按已结算商品形态展示，无操作按钮）。
- 时间范围为空 → 默认展示全部统计数据。
- 搜索 / 筛选无结果 → 显示空态文案"未找到匹配结果" + "清空筛选"按钮。
- 本模块**不提供统计报表导出**入口。

### 4.4 🟧 提示

| 场景 | 文案 |
|---|---|
| 无数据 | 暂无统计数据 |
| 搜索无结果 | 未找到匹配结果 |
| 加载失败 | 数据加载失败，请刷新后重试 |

## 5. 材料费用支出

材料费用用于统计平台在寄售、包装、处理商品等过程中产生的材料支出。

### 5.1 录入字段

| 字段 | 类型 | 必填 | 限制 |
|---|---|---|---|
| 支出项目 | 单选 / 自定义文本 | 是 | 预设：包装盒 / 防撞材料 / 透明袋 / 标签纸 / 胶带 / 其他耗材；选"其他耗材"时可输入自定义名称（上限 20 字） |
| 支出金额 | 数值输入 | 是 | 大于 0，最多 2 位小数 |
| 发生日期 | 日期选择器 | 是 | 默认今天，不能晚于今天 |
| 备注 | 文本输入 | 否 | 上限 200 字 |
| 凭证图片 | 图片上传 | 否 | 最多 3 张，单张 ≤ 5M |

### 5.2 🟩 业务规则

- 不同材料费用作为不同支出项目加入统计（不分摊到商品）。
- 只统计每笔材料支出金额，不与具体商品挂钩。
- 录入后可在记录列表中**编辑 / 删除**该笔支出，删除需二次确认。

### 5.3 🟦 显示规则

**列表项**

| 元素 | 内容 |
|---|---|
| 主标题 | 支出项目名称 |
| 副标题 | 发生日期 + 备注节选 |

## 7. 统计汇总关系

统计模块至少形成以下基础汇总：

### 7.1 基础汇总项

| 汇总项 | 公式 | 说明 |
|---|---|---|
| 已结算商品价格总额 | sum(商品价格 where 状态 = 已结算) | 寄售标价合计 |
| 已结算商品实际收入总额 | sum(商品实际收入 where 状态 = 已结算) | 管理员在外部平台真实卖到的钱合计 |
| 平台抽成总收入 | sum(商品价格 × 抽成比例 where 状态 = 已结算) | 抽成基数为商品价格 |
| 寄售用户结算总支出 | sum(商品价格 × (1 − 抽成比例) where 状态 = 已结算) | 即"应付给寄售用户的钱合计" |
| 平台差价收益 | 已结算商品实际收入总额 − 已结算商品价格总额 | 可正可负（赚 / 亏） |
| 材料费用总支出 | sum(材料支出) | — |
| 物流费用总支出 | sum(物流支出) | — |
| 总支出 | 材料费用总支出 + 物流费用总支出 | — |

### 7.2 平台净收入

**平台净收入 = 平台抽成总收入 + 平台差价收益 − 总支出**

展开后：

```text
平台净收入 = sum(商品价格 × 抽成比例)             ← 抽成收益
           + sum(商品实际收入 − 商品价格)         ← 差价收益（可正可负）
           − 材料费用总支出
           − 物流费用总支出
```

等价表达（用于核对）：

```text
平台净收入 = sum(商品实际收入)                    ← 平台实际收到的钱
           − sum(商品价格 × (1 − 抽成比例))      ← 付给寄售用户的钱
           − 材料费用总支出
           − 物流费用总支出
```

> 收入统计**只计入已结算商品**的金额，其他状态商品不计入。

🟦 显示规则：
- 汇总项以列表形式展示在统计页底部。
- "平台净收入"用主色高亮强调。
- "平台差价收益"为正时灰色显示，为负时红色显示（提示管理员存在亏损）。

## 8. 验收标准

### 8.1 统计首页

- Hero 主卡右侧需展示“查看明细”按钮，点击进入收入明细页并带入当前月份或累计筛选条件。
- 统计首页不再展示底部“实际收入（平台净收入）”蓝色汇总条。
- 统计首页需展示查询明细卡：实际销售额、结算金额、费用支出；卡片样式与图中一致，均带右侧箭头。
- 点击“费用支出”卡进入费用支出明细视图，展示材料费用与物流费用记录。
- 统计首页本月模式下展示“生成寄售月报”入口卡，文案为“生成寄售月报 / 一键生成宣传海报 · 保存分享拉新”。
- 点击“生成寄售月报”后进入寄售月报页，并默认带入统计首页当前选择月份。
- 累计模式下不允许生成月报。

### 8.2 寄售月报

- 月报页标题为“寄售月报”，副标题为“生成图片 · 用于宣传分享”。
- 月报页支持切换历史月份，切换后海报数据同步刷新。
- 海报需展示店铺信息、月报月份、本月成交金额、较上月趋势、成交件数、寄售 IP、平台总商品、当前在售、本月成交、寄售谷友、小程序用户、宣传 CTA、小程序码和生成日期。
- 用户端敏感财务口径不得出现在月报中；月报仅展示宣传口径数据，并标注“仅供参考”。
- 底部主按钮必须为“保存图片”，点击后保存海报到系统相册。
- 保存图片内容只包含海报区域，不包含管理端底部 TabBar、页面顶部状态栏和“保存图片”按钮。
- 保存成功提示“图片已保存到相册”；相册权限被拒绝时提示“请开启相册权限后再保存”。
