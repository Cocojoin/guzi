## 1. IP管理

### 1.1 页面目标
- 统一管理用户端“谷子集市”的 IP 分类。
- 支持平台手动新增 IP。
- 支持将商品分配到指定 IP、从 IP 中移除商品。
- 支持删除 IP。

### 1.2 设置首页入口
- 设置首页增加“IP管理”入口。
- 入口信息建议：
  - 标题：IP管理
  - 副标题：新增IP、调整商品归属、删除IP

### 1.3 数据口径与自动同步规则
- IP 管理以商品字段 `ip` 为主口径。
- IP 名称统一去除首尾空格后保存和比较。
- IP 列表由两类数据合并生成：
  - 商品中已有 `ip` 字段自动派生的 IP
  - 管理员手动新增、但暂时没有商品的空 IP
- 手动新增的空 IP 优先写入云数据库 `ip_groups`，若云端不可用则使用本地缓存兜底。
- 管理端每新增商品时，系统根据商品的 `ip` 字段自动同步对应 IP：
  - 若该 IP 不存在，则自动创建 IP。
  - 若该 IP 已存在，则该商品自动加入已有 IP。
- 管理端编辑商品时：
  - 若商品 IP 被修改，则商品应从旧 IP 迁移到新 IP。
  - 若新 IP 不存在，则自动创建。
- 批量新增商品时，同样适用上述自动同步规则。
- IP 名称按完整文本匹配，默认去除首尾空格后比较。

### 1.4 列表展示规则
- IP 管理列表每项至少展示：
  - IP 名称
  - 商品总数
  - 当前在售商品数
  - 当前角色数
  - IP 首字母 / 简称
  - 空 IP 标识（当商品总数为 0 时）
  - 最近更新时间（可选）
- 默认排序建议：
  1. 当前在售商品数倒序
  2. 最近更新时间倒序
  3. IP 名称正序

### 1.5 新增 IP
- 管理员可手动新增 IP。
- 字段：
  - IP 名称（必填）
- 校验规则：
  - 不能为空
  - 去除首尾空格后存储
  - 不允许与现有 IP 重名
- 新增成功后立即刷新列表。
- 新增空 IP 后，即使没有商品，也在管理端列表显示。
- 空 IP 不影响用户端；用户端只展示有上架商品的 IP。

### 1.5.1 IP详情页
- 点击 IP 卡片进入 IP 详情页。
- 页面展示：
  - IP 名称
  - 商品总数
  - 当前在售商品数
  - 当前角色数
  - 当前 IP 下商品列表
- 商品列表按更新时间倒序展示。
- 点击商品卡片进入管理端商品详情。
- 支持勾选商品、全选 / 取消全选。
- 未选择商品时点击“移出”提示"请先选择商品"。

### 1.6 分配商品到 IP
- 在某个 IP 下支持“添加商品 / 分配商品”操作。
- 可选择 1 个或多个商品加入该 IP。
- 被分配商品的 `ip` 字段直接更新为目标 IP 名称。
- 若商品原本属于其他 IP，则视为迁移，不保留在原 IP 中。
- 分配商品页展示可分配商品列表，支持按商品编号、标题、角色、系列、当前 IP 搜索。
- 已属于目标 IP 的商品不可再次选择。
- 筛选 / 搜索后需保留仍可见且可选的选择状态。
- 分配成功后：
  - 当前 IP 商品数实时更新
  - 原 IP 商品数同步更新

### 1.7 从 IP 中移除商品
- 在某个 IP 下支持“移除商品”操作。
- 可选择 1 个或多个当前属于该 IP 的商品移除。
- 移除后，商品的 `ip` 字段清空。
- 被移除商品不会自动删除。
- 若商品已上架但被移除 IP，则该商品不再出现在用户端任何 IP 分类和对应列表中。
- 移出商品需二次确认："确认将所选 N 件商品移出该 IP 吗？"

### 1.8 删除 IP
- IP 支持删除。
- 删除规则：
  - 仅允许删除“IP 内无商品”的空 IP
  - 只要该 IP 下仍存在商品，就不允许删除
  - 删除成功后，该 IP 不再出现在管理端和用户端
- 删除操作必须二次确认。
- 当 IP 下仍有商品时，删除按钮置灰或点击后直接提示不可删除原因。
- 删除成功后返回 IP 管理列表。

### 1.9 用户端联动规则
- 用户端谷子集市首页仅展示“有上架商品”的 IP。
- 用户端 Banner 仅展示最近上新的上架 IP。
- 用户端点击某个 IP 后，进入该 IP 的商品列表页。
- 若某个 IP 被删除或其下商品全部不再上架，则该 IP 不再出现在用户端。

### 1.10 操作日志
- 以下操作需写入操作日志：
  - 新增 IP
  - 删除 IP
  - 分配商品到 IP
  - 从 IP 中移除商品
  - 商品编辑导致 IP 迁移
  - 商品新增自动创建 IP（可简化记入“新增商品”备注）

### 1.10.1 权限与异常
- IP 管理仅管理员可访问。
- 非管理员进入 IP 管理、IP 详情、分配页时，重定向到登录页。
- IP 数据加载失败时 Toast "IP 数据加载失败" 或 "IP 详情加载失败"。
- 商品分配列表加载失败时 Toast "商品数据加载失败"。
- 云数据库 `ip_groups` 不存在或无权限时，手动 IP 可先落本地缓存兜底；需要在部署阶段补齐集合与权限。

### 1.11 提示规则

| 场景 | 文案 |
|---|---|
| 新增 IP 成功 | IP 已新增 |
| IP 名称为空 | 请输入 IP 名称 |
| IP 重名 | IP 已存在，请勿重复添加 |
| 分配商品成功 | 已加入该 IP |
| 移除商品成功 | 已移出该 IP |
| 删除 IP 成功 | IP 已删除 |
| 删除受限 | 当前 IP 下仍有商品，无法删除 |
| 删除确认 | 确认删除该空 IP 吗？ |
| IP 列表加载失败 | IP 数据加载失败 |
| IP 详情加载失败 | IP 详情加载失败 |
| 可分配商品加载失败 | 商品数据加载失败 |
| 未选择商品 | 请先选择商品 |

## 2. 店铺信息设置

### 2.1 页面目标

- 统一维护用户端“在哪里买”模块展示的官方店铺信息。
- 避免管理员在每个商品中重复填写店铺名称。
- 支持为淘宝/天猫、拼多多、小红书分别配置店铺名称、搜索关键词和展示状态。

### 2.2 设置首页入口

- 设置首页增加“店铺信息设置”入口。
- 入口信息建议：
  - 标题：店铺信息设置
  - 副标题：维护淘宝、拼多多、小红书店铺名称

### 2.3 字段规则

| 平台 | 是否固定 | 字段 | 规则 |
|---|---|---|---|
| 淘宝/天猫 | 是 | 店铺名称 | 1-30 字，用户端展示 |
| 淘宝/天猫 | 是 | 搜索关键词 | 1-30 字；为空时默认使用店铺名称 |
| 淘宝/天猫 | 是 | 是否启用 | 开启后商品可在该平台展示购买途径 |
| 拼多多 | 是 | 店铺名称 | 1-30 字，用户端展示 |
| 拼多多 | 是 | 搜索关键词 | 1-30 字；为空时默认使用店铺名称 |
| 拼多多 | 是 | 是否启用 | 开启后商品可在该平台展示购买途径 |
| 小红书 | 是 | 店铺名称 | 1-30 字，用户端展示 |
| 小红书 | 是 | 搜索关键词 | 1-30 字；为空时默认使用店铺名称 |
| 小红书 | 是 | 是否启用 | 开启后商品可在该平台展示购买途径 |

🟩 业务规则：
- 三个平台为固定配置项，不支持管理员新增自定义平台。
- 店铺名称是全局配置，同一平台下所有商品统一使用该店铺名称。
- 搜索关键词用于用户端复制；如果未填写，则复制店铺名称。
- 平台关闭后，商品编辑页不可选择该平台作为用户端购买途径；已配置该平台的商品在用户端不展示该平台。
- 至少建议启用 1 个平台；如全部关闭，用户端商品详情不展示“在哪里买”模块。

### 2.4 交互规则

- 页面以 3 张平台配置卡展示：淘宝/天猫、拼多多、小红书。
- 每张卡包含启用开关、店铺名称输入框、搜索关键词输入框。
- 点击“保存”后统一保存三个平台配置。
- 保存成功后，用户端商品详情页读取最新店铺名称和搜索关键词。
- 若商品编辑页已选择某平台，修改该平台店铺名称后，用户端无需逐个商品更新，自动展示新名称。

### 2.5 校验与提示

| 场景 | 文案 |
|---|---|
| 保存成功 | 店铺信息已保存 |
| 保存失败 | 保存失败，请稍后重试 |
| 启用平台但店铺名称为空 | 请填写店铺名称 |
| 店铺名称超长 | 店铺名称不能超过 30 个字 |
| 搜索关键词超长 | 搜索关键词不能超过 30 个字 |

### 2.6 操作日志

- 保存店铺信息设置需写入操作日志。
- 日志记录变更平台、变更字段、操作人、操作时间。

## 3. 操作日志

### 3.1 显示内容

| 字段 | 显示 |
|---|---|
| 操作时间 | YYYY-MM-DD HH:mm:ss |
| 操作人 | 用户昵称 + 用户 ID |
| 操作类型 | 见 §2.3 操作类型枚举 |
| 操作对象 | 商品编号 / 用户 ID / 记录 ID 等 |
| 操作结果 | 成功 / 失败 |
| 备注 | 如失败原因 |
| 异常行为标记 | 异常行为显示红色"!"图标，便于排查 |

### 3.2 筛选与查询
- 支持关键词搜索，范围包含：操作标题、操作对象、操作人、备注。
- 支持按周期筛选：本月 / 近三月 / 全部。
- 支持按操作类型筛选：全部类型 + 当前日志中出现过的类型。
- 支持按操作人筛选：全部人员 + 当前日志中出现过的操作人。
- 支持按操作结果筛选：全部结果 / 仅成功 / 仅失败。
- 筛选条件即时生效，多个条件取交集。

### 3.3 操作类型枚举（最小集合）

| 类型 | 说明 |
|---|---|
| 上传商品 | 单个 / 批量 |
| 编辑商品 | 字段变更明细 |
| 删除商品 | 单个 / 批量 |
| 修改商品状态 | 流转前状态 → 流转后状态 |
| 上架并填链接 | 单个 / 批量 |
| 提交结算 | 含商品清单 |
| IP管理 | 新增IP / 删除IP / 分配商品 / 移除商品 |
| 重置密码 | 目标用户 |
| 修改抽成 | 原值 → 新值 |
| 修改寄售权限 | 开 / 关 |
| 编辑用户信息 | 字段变更明细 |
| 店铺信息设置 | 店铺名称 / 搜索关键词 / 启用状态变更 |
| 导出数据 | 范围 / 范围说明 |
| 数据备份 | 自动 / 手动 |
| 登录 / 登出 | 异常登录标记为异常行为 |

### 3.4 提示规则

| 场景 | 文案 |
|---|---|
| 列表为空 | 暂无操作记录 |
| 加载失败 | 数据加载失败，请刷新后重试 |
| 修改保留周期成功 | 已改为[保留周期] |
| 清理过期日志成功 | 已清理 N 条 / 暂无过期日志 |

### 3.5 日志保留规则

| 项 | 规则 |
|---|---|
| 默认保留周期 | 90 天 |
| 可选保留周期 | 7 天 / 30 天 / 90 天 / 180 天 / 永久保留 |
| 清理方式 | 进入日志页时按当前保留周期自动清理；管理员也可点击"清理过期日志"手动执行 |
| 存储口径 | 优先写入云数据库 `admin_operation_logs`，同时保留本地缓存兜底 |
| 本地缓存上限 | 最近 300 条 |

## 5. 导出数据

| 元素 | 显示 |
|---|---|
