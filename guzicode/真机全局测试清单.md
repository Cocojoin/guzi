# 真机全局测试清单

## 一、环境准备

1. 微信开发者工具已切到目标云开发环境。
2. 已上传并部署云函数：`auth`。
3. 数据库已存在 `users`、`products` 集合，且管理员账号可用。
4. 真机测试前清空小程序缓存，确保不是旧登录态干扰。
5. `users` 集合权限已收紧为“前端不可直接读写，仅云函数访问”。

## 二、认证链路

1. **普通用户注册**
   - 新账号注册成功，自动跳转资料页。
   - 重复账号注册时，页面提示“账号已存在”。
   - 弱格式账号 / 密码 / 二次确认不一致时，前端校验正确拦截。

2. **普通用户登录**
   - 新注册账号可正常登录。
   - 错误密码提示正确，不误跳页。
   - 不存在账号提示正确。
   - 被禁用账号无法登录。

3. **首次资料保存**
   - 注册后填写昵称、上传头像可保存成功。
   - 昵称非法字符、超长、为空时拦截。
   - 上传超 5M 图片时前端拦截。

4. **修改密码**
   - 原密码正确时可修改成功，成功后强制回登录页。
   - 原密码错误时提示正确。
   - 新旧密码相同、确认密码不一致、长度不合法时拦截。

5. **管理员重置密码**
   - 管理员在用户管理页可将目标用户密码重置为 `123456`。
   - 被重置用户可用新密码登录，旧密码失效。

6. **登录态失效**
   - 登录后清缓存或更换微信号，再执行改密 / 保存资料，应提示重新登录。

## 三、角色与页面跳转

1. 管理员登录后进入管理端首页。
2. 普通用户登录后进入用户端首页。
3. 未登录访问需要登录的页面时，能正确回到登录页。
4. 退出登录后再次进入“我的”相关页面，不应残留旧信息。

## 四、用户侧核心流程

1. 商品列表可正常加载。
2. “我的”页面能正常显示昵称、账号、头像。
3. 寄售入口对无权限用户提示正常。
4. 有寄售权限用户可进入寄售相关页面。

## 五、管理侧核心流程

1. 商品列表、详情、上传、编辑可正常进入。
2. 用户列表可正常加载。
3. 开关寄售权限、查看用户详情、重置密码流程可用。
4. 管理日志 / 操作提示文案正常，没有明显报错。

## 六、异常回归

1. 弱网下登录、注册、保存资料时，失败提示可理解。
2. 云函数未部署时，页面会提示先上传 `auth` 云函数。
3. 云环境切错时，登录链路能暴露明确错误，不出现静默失败。

## 七、上线前再确认

1. `users` 集合权限已收紧，避免前端直接读写敏感字段。
2. 管理员初始密码已改为正式值。
3. 真机覆盖 iOS / Android 各至少 1 台。
4. 用新用户、老用户、管理员三个身份各跑一遍关键流程。
